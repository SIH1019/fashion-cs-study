# Fashion CS Study

컴퓨터공학 기초를 다시 다지고, 패션 추천 시스템을 만들기 위한 학습 기록 저장소입니다.

---

## 1. 저장소 목적

- 컴퓨터공학 기초를 다시 공부하기 위한 저장소
- 공부한 내용을 실제 프로젝트에 적용하기 위한 저장소
- 패션 추천 시스템 개발 과정을 기록하기 위한 저장소

- 최종 목표
  - 색상, 핏, 기장, 상황 등을 분석하는 패션 추천 시스템 제작
  - 옷 조합을 추천하는 것에서 끝나지 않고, 왜 어울리는지 설명하는 시스템 제작
  - 사용자가 코디의 문제점과 개선 방향을 이해할 수 있는 서비스 제작

- 복학 전 1차 목표
  - 완성형 앱을 바로 만드는 것이 목표가 아님
  - Java 기본기와 객체지향 개념을 먼저 다지는 것이 목표
  - 작은 콘솔 프로그램으로 패션 추천 로직을 구현해보는 것이 목표

---

## 2. 학습 목표

- Java 기본 문법 복습
- 객체지향 개념 정리
- 패션 도메인 지식 정리
- 패션 지식을 규칙과 알고리즘으로 변환
- 패션 추천 콘솔 프로그램 제작
- 이후 SwiftUI 기반 iOS 앱으로 확장

---

## 3. 전체 진행 순서

- 1단계: 왜 만드는지 정의
- 2단계: 누구를 위한 서비스인지 정하기
- 3단계: 기존 서비스와 차별점 분석
- 4단계: 웹/앱 중 무엇으로 시작할지 결정
- 5단계: 언어와 기술 스택 결정
- 6단계: 패션 도메인 지식 공부
- 7단계: 패션 기준을 알고리즘화
- 8단계: 핵심 기능 정하기
- 9단계: 작은 버전으로 구현
- 10단계: 서버/DB 연결
- 11단계: 배포
- 12단계: 유지보수 및 개선

---

## 4. 프로젝트 방향

- 단순히 옷을 추천하는 앱이 아님
- 옷 조합을 논리적으로 분석하는 시스템을 목표로 함
- 사용자가 왜 이 조합이 어울리는지 이해할 수 있도록 설명하는 것이 핵심
- 감각에만 의존하는 패션 추천이 아니라, 기준과 규칙이 있는 추천 시스템을 만들고자 함

- 분석 요소
  - 색상
  - 명도
  - 채도
  - 핏
  - 기장
  - 상황
  - 분위기
  - 개선 방향

---

## 5. 추천 결과 예시

- 입력 예시
  - 상의: 네이비
  - 하의: 블랙
  - 신발: 화이트

- 출력 예시

```text
네이비와 블랙은 차분하고 시크한 조합입니다.
다만 상하의가 모두 어두운 계열이라 전체적으로 답답해 보일 수 있습니다.
화이트 신발이나 밝은 이너를 추가하면 명도 대비가 생겨 균형이 좋아집니다.
```

- 목표
  - 단순 추천이 아니라 추천 이유 제공
  - 조합의 장점과 단점 분석
  - 개선 방법까지 제안
