# 패션 추천 시스템 최종 기획 정리

## 1. 내가 만들려는 서비스

- 단순히 옷끼리 잘 어울리는지 추천해주는 앱이 아님
- 색상뿐만 아니라 핏, 기장, 실루엣, 상황까지 함께 고려하는 패션 추천 시스템
- 패션을 감각적으로만 판단하는 것이 아니라, 논리와 기준으로 분석하는 서비스
- 사용자가 옷 조합을 입력하면 왜 어울리는지, 왜 어색한지, 어떻게 고치면 좋은지 설명해주는 앱
- 최종적으로 사용자가 혼자서도 스타일링할 수 있도록 돕는 패션 학습형 추천 서비스

---

## 2. 왜 만드는가

- 옷을 잘 입고 싶지만 처음부터 어떻게 입어야 할지 감이 잡히지 않는 사람이 많음
- 옷을 사도 어떻게 조합해야 할지 모르면 활용도가 떨어짐
- 기존 추천 서비스는 결과를 보여주는 경우가 많지만, 사용자가 패션 원리를 이해하기는 어려움
- 패션 초보자가 코디의 기준을 배우고, 나중에는 스스로 판단할 수 있도록 돕고 싶음
- 쇼핑몰에서 본 옷이나 사용자가 선택한 옷 조합을 가져왔을 때, 왜 어울리는지 또는 왜 어색한지 분석해주고 싶음

---

## 3. 누구를 위한 서비스인가

- 패션에 입문하려는 사람
- 옷을 잘 입고 싶지만 감각적으로 판단하기 어려운 사람
- 코디를 봐도 왜 예쁜지, 왜 어색한지 설명하기 어려운 사람
- 쇼핑몰에서 본 옷을 어떻게 조합해야 할지 모르는 사람
- 감각보다 논리, 공식, 기준으로 이해하는 것이 편한 사람
- 특히 공대생처럼 패션을 구조적으로 이해하고 싶은 사람

---

## 4. 도메인 지식 활용 방향

- 본 서비스의 핵심은 패션 도메인 지식을 사용하는 것
- 도메인 지식이란 특정 분야에서 사용되는 전문적인 기준과 지식을 의미함
- 이 프로젝트에서의 도메인 지식은 패션 분야의 색상 조합, 명도, 채도, 핏, 기장, 실루엣, 상황별 옷차림 기준 등을 의미함

- 단순히 AI에게 “옷 추천해줘”라고 맡기는 방식이 아님
- 직접 패션을 공부하고, 패션에서 사용되는 판단 기준을 정리할 예정
- 정리한 기준을 코드로 구현할 수 있도록 규칙화할 예정
- 패션 감각을 추상적인 감으로만 두지 않고, 알고리즘으로 바꾸는 것이 목표

- 도메인 지식 활용 흐름
  - 패션 이론 및 스타일링 기준 공부
  - 색상, 핏, 기장, 상황별 판단 기준 정리
  - 기준을 조건문, 점수, 규칙으로 변환
  - Java 콘솔 프로그램으로 1차 구현
  - 이후 앱 서비스로 확장

---

## 5. 도메인 지식을 기반으로 한 알고리즘 제작 방향

- 본 프로젝트의 핵심은 내가 직접 패션 도메인 지식을 공부한 뒤, 그 내용을 바탕으로 나만의 추천 알고리즘을 만드는 것

- 단순히 기존 AI 모델이나 추천 기능에 의존하는 것이 아니라, 패션에서 실제로 사용되는 판단 기준을 직접 정리할 예정

- 공부할 도메인 지식
  - 색상 조합
  - 명도와 채도
  - 톤의 통일감
  - 포인트 컬러 사용
  - 상의와 하의의 핏 균형
  - 상의 기장과 하의 허리선의 관계
  - 전체 실루엣
  - 상황별 옷차림 기준
  - 소재와 분위기
  - 신발, 가방, 액세서리의 역할

- 공부한 내용을 단순히 문서로 정리하는 것에서 끝내지 않고, 코드로 판단할 수 있는 규칙으로 바꿀 예정

- 예시
  - 어두운 색끼리 조합되면 답답해 보일 수 있음
  - 이 기준을 알고리즘에서는 `상의 명도`와 `하의 명도`가 모두 낮을 때 감점하는 방식으로 구현 가능
  - 대신 신발이나 가방에 밝은 색이 있으면 포인트 컬러로 판단하여 점수를 보완할 수 있음

- 또 다른 예시
  - 상의가 오버핏이고 하의도 와이드핏이면 전체 실루엣이 무거워질 수 있음
  - 이 기준을 알고리즘에서는 `상의 핏 = 오버핏`, `하의 핏 = 와이드핏`일 때 실루엣 균형 점수를 낮추는 방식으로 구현 가능
  - 대신 상의 기장이 짧거나 허리선이 드러나면 비율 보정 요소로 판단할 수 있음

- 알고리즘 제작 흐름
  - 패션 도메인 지식 공부
  - 판단 기준 정리
  - 기준을 조건문과 점수 체계로 변환
  - Java 콘솔 프로그램으로 1차 구현
  - 결과에 대한 설명 문장 생성
  - 이후 앱 서비스로 확장

- 최종 목표
  - 사용자가 입력한 코디를 색상, 핏, 기장, 상황 기준으로 분석
  - 각 항목별 점수 제공
  - 점수가 나온 이유 설명
  - 어색한 부분과 개선 방향 제안
  - 사용자가 패션 원리를 학습하도록 돕기

## 6. 변별력을 만들기 위해 해야 하는 것

### 6-1. 단순 추천에서 끝나지 않기

- “이 옷이 어울립니다”만 보여주면 기존 서비스와 차별점이 약함
- 추천 결과뿐 아니라 추천 이유를 반드시 설명해야 함
- 점수, 이유, 문제점, 개선 방향을 함께 제공해야 함

---

### 6-2. 도메인 지식을 알고리즘화하기

- 패션 도메인 지식을 직접 공부해야 함
- 색상, 핏, 기장, 상황별 기준을 정리해야 함
- 정리한 기준을 코드로 판단할 수 있도록 알고리즘화해야 함
- 어떤 기준 때문에 점수가 높거나 낮은지 설명할 수 있어야 함

---

### 6-3. 사용자가 배우게 만들기

- 기존 앱처럼 AI가 대신 코디를 골라주는 것에서 끝나면 안 됨
- 사용자가 코디의 원리를 이해하도록 해야 함
- 분석 결과 마지막에 패션 원리나 개선 포인트를 제공해야 함
- 사용자가 앱을 사용할수록 혼자 스타일링할 수 있도록 돕는 구조가 필요함

---

## 8. 기존 앱과 내 서비스의 핵심 차이

- 결과 중심의 추천이 많음
- 사용자가 왜 그 조합이 좋은지 깊게 이해하기 어려움

### 내 서비스의 차별점

- 단순히 옷끼리의 매치만 보지 않음
- 색상, 핏, 기장, 실루엣, 상황을 함께 분석함
- 패션 도메인 지식을 직접 공부하고 기준화함
- 기준을 바탕으로 나만의 알고리즘을 만듦
- 추천 결과뿐 아니라 이유와 개선 방향을 설명함
- 사용자가 앱을 사용할수록 패션 원리를 배우게 함
- 최종 목표는 사용자가 혼자 스타일링할 수 있도록 돕는 것

## 9. 알고리즘으로 만들어야 할 기준

### 9-1. 색상 알고리즘

- 옷의 색상을 RGB 또는 HEX 값으로 표현
- 색상을 색상 계열, 명도, 채도 등으로 분리
- 상의와 하의의 색상 조합 판단
- 전체적으로 너무 어두운지, 너무 튀는지 판단
- 포인트 컬러가 있는지 판단
- 색상 대비가 적절한지 판단

예시

- 네이비 + 블랙
  - 차분하고 시크한 조합
  - 하지만 전체적으로 어두워 답답해 보일 수 있음
  - 화이트 신발이나 밝은 이너를 추가하면 균형이 좋아짐

---

### 9-2. 핏 알고리즘

- 상의 핏이 슬림핏인지, 정핏인지, 오버핏인지 판단
- 하의 핏이 슬림핏인지, 스트레이트핏인지, 와이드핏인지 판단
- 상의와 하의의 실루엣 균형 판단
- 상하의가 모두 크면 전체적으로 부해 보일 수 있음
- 상의가 짧고 하의가 하이웨이스트면 다리가 길어 보일 수 있음

예시

- 크롭 상의 + 하이웨이스트 하의
  - 허리선이 올라가 보여 다리가 길어 보임

- 오버핏 상의 + 와이드 팬츠
  - 편안한 느낌은 있지만 전체적으로 무거워 보일 수 있음

---

### 9-3. 기장 알고리즘

- 상의 기장이 짧은지, 보통인지, 긴지 판단
- 하의 허리선과 상의 기장의 관계 판단
- 하의 길이가 신발을 덮는지 판단
- 치마나 바지 길이가 전체 비율에 미치는 영향 분석
- 다리가 길어 보이는 조합인지 판단
- 상체와 하체 비율이 어색해지는 조합인지 판단

---

### 9-4. 상황 알고리즘

- 사용자가 입으려는 상황을 입력
- 데일리룩, 학교, 데이트, 발표, 출근, 격식 있는 자리 등으로 구분
- 상황에 비해 너무 캐주얼한지 판단
- 상황에 비해 너무 과한지 판단
- 상황별 적합도를 점수화

예시

- 데이트룩
  - 너무 편한 운동복 느낌이면 감점
  - 깔끔한 색 조합과 적당한 포인트가 있으면 가점

- 발표룩
  - 너무 캐주얼한 신발이나 과한 노출은 감점
  - 단정한 실루엣과 차분한 색감은 가점

---

### 9-5. 개선 제안 알고리즘

- 코디에서 가장 어색한 요소를 찾음
- 한 번에 너무 많은 수정을 제안하지 않음
- 가장 효과적인 수정 방향 하나를 우선 제안
- 색상이 답답하면 밝은 포인트 제안
- 핏이 무거우면 상의 기장이나 하의 실루엣 조정 제안
- 상황에 맞지 않으면 신발, 가방, 아우터 변경 제안

## 10. 핵심 기능

### 10-1. 코디 입력 기능

- 색상, 핏, 기장, 상황 정보를 함께 입력

- 전체 코디 점수 계산

### 10-3. 이유 설명 기능

### 10-4. 개선 제안 기능

  - 상의 기장 조정
  - 하의 핏 변경

  - 상황에 맞는 신발이나 가방 추천

### 10-5. 학습 기능

## 12. 최종 한 줄 정의

본 서비스는 단순한 옷 매치 추천 앱이 아니라, 색상·핏·기장·상황을 패션 도메인 지식 기반 알고리즘으로 분석하여 사용자가 코디의 원리를 이해하고 스스로 스타일링할 수 있도록 돕는 논리 기반 패션 학습형 추천 서비스이다.

## 서비스 필요성 및 타깃 근거

### 1. 서비스 기획 배경

- 본 서비스는 단순히 옷을 추천해주는 앱이 아니라, 패션 초보자가 코디의 기준을 이해하고 스스로 스타일링할 수 있도록 돕는 서비스
- 옷을 잘 입고 싶지만 처음부터 어떻게 입어야 할지 감이 잡히지 않는 사람을 주요 대상
- 기존 패션 앱은 주로 옷을 추천하거나 판매하는 데 초점이 있지만, 사용자가 왜 그 조합이 어울리는지 학습하기는 어령ㅁ
- 따라서 본 서비스는 패션 도메인 지식을 기반으로 색상, 핏, 기장, 실루엣, 상황별 기준을 알고리즘화하여 설명하는 방향으로 기획함

---

### 2. 주요 타깃 사용자

- 20대 대학생
- 20대 사회초년생
- 패션에 입문하려는 사람
- 온라인 쇼핑몰에서 옷을 자주 보지만 어떻게 조합해야 할지 모르는 사람
- 옷을 사고도 활용을 잘 못하는 사람
- 코디를 봐도 왜 예쁜지, 왜 어색한지 설명하기 어려운 사람
- 감각보다 논리, 기준, 공식으로 이해하는 것이 편한 사람
- 공대생처럼 패션을 구조적으로 이해하고 싶은 사람

- 20대 남성도 무신사 등 패션 플랫폼 이용이 활발하므로 패션 입문자 타깃으로 적합함
- 따라서 초기 서비스 타깃은 “20대 남녀 패션 초보자”로 설정하는 것이 적절함

### 7. 도메인 지식을 기반으로 한 알고리즘 제작 방향

- 본 프로젝트의 핵심은 내가 직접 패션 도메인 지식을 공부한 뒤, 그 내용을 바탕으로 나만의 추천 알고리즘을 만드는 것
- 단순히 기존 AI 모델이나 추천 기능에 의존하지 않음
- 패션에서 실제로 사용되는 판단 기준을 직접 정리하고, 이를 코드로 구현할 수 있는 규칙으로 바꿈

#### 공부할 도메인 지식

- 색상 조합
- 명도와 채도
- 톤의 통일감
- 포인트 컬러 사용
- 상의와 하의의 핏 균형
- 상의 기장과 하의 허리선의 관계
- 전체 실루엣
- 상황별 옷차림 기준
- 소재와 분위기
- 신발, 가방, 액세서리의 역할

#### 알고리즘화 예시

- 상의와 하의가 모두 어두운 색이면 답답해 보일 수 있으므로 감점
- 신발이나 가방에 밝은 색이 있으면 포인트 컬러로 판단하여 가점
- 상의가 슬림핏이고 하의가 와이드핏이면 실루엣 균형이 좋아 가점
- 상의와 하의가 모두 오버핏이면 전체적으로 부해 보일 수 있으므로 감점
- 크롭 상의와 하이웨이스트 하의 조합은 비율이 좋아 보일 수 있으므로 가점
- 발표나 격식 있는 자리에서 너무 캐주얼한 아이템이 많으면 감점

### 8. 최종 서비스 정의

본 서비스는 단순한 코디 추천 앱이 아니라, 패션 도메인 지식을 직접 공부하고 기준화하여 색상·핏·기장·상황별 판단 기준을 알고리즘으로 구현하는 논리 기반 패션 학습형 추천 서비스이다.

사용자가 옷 조합을 입력하면 단순히 어울린다/어울리지 않는다를 말하는 것이 아니라, 왜 어울리는지, 왜 어색한지, 어떤 요소를 바꾸면 더 좋아지는지를 설명한다.

최종 목표는 사용자가 앱에만 의존하는 것이 아니라, 패션 원리를 이해하고 스스로 스타일링할 수 있도록 돕는 것이다.
