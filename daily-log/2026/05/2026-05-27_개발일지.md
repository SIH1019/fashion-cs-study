## 6. 오늘 알게 된 점

- Mac에서는 Git Bash를 따로 설치하지 않아도 Terminal에서 Git 명령어 사용 가능
- `git --version`으로 Git 설치 여부 확인 가능
- `git init`은 현재 폴더를 Git 저장소로 만드는 명령어
- `README.md`는 GitHub 저장소의 소개 문서 역할
- 개발일지는 날짜별로 작성하면 학습 과정을 추적하기 쉬움
- `code .` 명령어가 안 될 경우 VSCode PATH 설정이 되어 있지 않은 상태
- VSCode는 `open -a "Visual Studio Code" .` 명령어로도 실행 가능
- GitHub는 비밀번호 대신 Personal Access Token으로 인증해야 함
- API Key 같은 secret 파일이 포함되면 GitHub Push Protection으로 업로드가 차단될 수 있음
- `.gitignore`를 사용하면 업로드하면 안 되는 파일을 Git 추적 대상에서 제외할 수 있음

## 7. 발생한 문제 및 해결

### 7-1. `code .` 명령어 실행 오류

- 발생한 문제
  - `code .` 입력 시 `command not found` 오류 발생

- 원인
  - VSCode의 `code` 명령어가 PATH에 등록되어 있지 않았음

- 해결
  - `open -a "Visual Studio Code" .` 명령어로 VSCode 실행

---

### 7-2. GitHub 인증 오류

- 발생한 문제
  - GitHub 비밀번호로 push 시도 시 인증 실패

- 원인
  - GitHub는 Git 작업에서 계정 비밀번호 인증을 지원하지 않음
  - Personal Access Token이 필요함

- 해결
  - GitHub에서 Personal Access Token 생성
  - Password 입력 위치에 토큰을 붙여넣어 인증

---

### 7-3. Push Protection 오류

- 발생한 문제
  - GitHub push 과정에서 secret 감지 오류 발생

- 원인
  - 이전 Git 기록에 `.streamlit/secrets.toml` 파일이 포함되어 있었음
  - 해당 파일 안에 API Key로 보이는 값이 감지됨

- 해결
  - `.gitignore` 파일 생성
  - `.streamlit/`, `*.env`, `secrets.toml` 제외 설정 추가
  - 기존 Git 기록 초기화 후 다시 commit/push 진행
