5. 오늘 배운 개념

5-1. Git

Git은 코드와 문서의 변경 기록을 관리하는 도구이다.

Git을 사용하면 내가 어떤 파일을 만들었는지, 어떤 내용을 수정했는지 기록할 수 있다.  
나중에 이전 상태로 돌아가거나, 프로젝트가 어떻게 진행되었는지 확인할 수 있다.

오늘은 Git이 설치되어 있는지 확인하고, `git init` 명령어로 현재 폴더를 Git 저장소로 만들었다.

---

5-2. GitHub

GitHub는 Git으로 관리한 파일을 온라인에 올려 저장하고 공유할 수 있는 서비스이다.

앞으로 Java 공부 내용, 개발일지, 패션 도메인 지식, 프로젝트 코드를 GitHub에 기록할 예정이다.

GitHub를 사용하는 이유는 다음과 같다.

- 공부한 내용을 기록하기 위해
- 프로젝트 진행 과정을 남기기 위해
- 나중에 포트폴리오처럼 활용하기 위해
- 내가 어떤 방식으로 성장했는지 확인하기 위해

---

5-3. Markdown

Markdown은 문서를 보기 좋게 정리하는 문법이다.

GitHub에서는 `README.md`나 개발일지 파일을 Markdown 형식으로 작성하면 제목, 목록, 코드블록 등을 깔끔하게 보여준다.

오늘 만든 개발일지 파일도 Markdown 형식으로 작성한다.

예시:

```markdown
# 제목

## 소제목

- 목록 1
- 목록 2

```java
System.out.println("Hello Java");
```
```
