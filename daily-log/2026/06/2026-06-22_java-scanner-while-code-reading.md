# 2026-06-22 Java Scanner, while(true), break, 코드 독해 정리

## 1. 오늘 공부한 내용

오늘은 Java 입문에서 `Scanner`와 `while(true)`를 사용하는 입력 반복 문제를 다시 공부했다.

오늘 공부한 핵심 내용은 다음과 같다.

* `while(true)`를 사용하는 이유
* 반복 횟수가 정해진 문제와 정해지지 않은 문제의 차이
* `break`로 반복문을 종료하는 구조
* `"종료"`를 입력하면 끝나는 프로그램
* `-1`을 입력하면 끝나는 프로그램
* `Scanner`로 문자열과 숫자를 입력받는 방법
* `nextInt()` 뒤에 `nextLine()`을 한 번 더 쓰는 이유
* `sum`, `count`, `input` 변수의 역할
* 합계와 평균을 구하는 흐름
* `-1`은 계산할 숫자가 아니라 종료 신호라는 점
* 문제를 코드로 바꾸는 독해 방법
* 코드를 한 줄씩 읽는 연습

오늘 제일 크게 느낀 점은 내가 문법을 아예 모르는 것이 아니라,
문제와 코드를 읽고 흐름을 해석하는 코드 독해가 아직 부족하다는 점이었다.

## 2. 오늘 헷갈렸던 부분

오늘 헷갈렸던 부분은 다음과 같다.

```text
1. 이름과 나이를 한 번씩 입력받는데 왜 while(true)를 쓰는지 모르겠음
2. while(true)가 무한 반복이라는데 왜 필요한지 모르겠음
3. 사용자가 몇 번 입력할지 모른다는 말이 잘 와닿지 않음
4. -1을 입력하면 종료하는 코드가 왜 그렇게 흐르는지 모르겠음
5. sum = 0, count = 0, input = 0의 역할이 헷갈림
6. if (input == -1)이 왜 sum += input보다 먼저 나오는지 헷갈림
7. 코드를 보면 문법은 아는데 전체 독해가 어려움
```

오늘 정리한 결론은 다음이다.

```text
while(true)는 진짜 영원히 반복하려는 것이 아니라,
반복 횟수를 미리 모를 때 일단 계속 반복하다가
종료 조건을 만나면 break로 빠져나오게 하려고 쓰는 구조이다.
```

## 3. while(true)를 쓰는 이유

`while(true)`는 반복 횟수를 미리 모를 때 사용한다.

예를 들어 다음과 같은 문제에서는 몇 번 반복할지 알 수 없다.

```text
사용자가 "종료"를 입력할 때까지 반복한다.
사용자가 -1을 입력할 때까지 반복한다.
사용자가 그만하겠다고 할 때까지 반복한다.
```

이런 문제는 반복 횟수가 정해져 있지 않다.

사용자가 한 번만 입력하고 끝낼 수도 있고,
열 번 입력하고 끝낼 수도 있다.

그래서 다음처럼 작성한다.

```java
while (true) {
    // 계속 반복할 코드

    if (종료 조건) {
        break;
    }
}
```

이 구조의 뜻은 다음과 같다.

```text
1. 일단 계속 반복한다.
2. 반복문 안에서 입력을 받는다.
3. 입력값이 종료 조건인지 확인한다.
4. 종료 조건이면 break로 반복문을 끝낸다.
5. 종료 조건이 아니면 나머지 코드를 실행한다.
6. 다시 반복문의 처음으로 돌아간다.
```

---

## 4. for문과 while(true)의 차이

`for`문은 반복 횟수가 정해져 있을 때 사용하기 좋다.

예를 들어 다음 문제는 반복 횟수가 정해져 있다.

```text
학생 5명의 점수를 입력하세요.
```

이 경우에는 5번 반복하면 된다.

```java
for (int i = 0; i < 5; i++) {
    // 5번 반복
}
```

하지만 다음 문제는 반복 횟수가 정해져 있지 않다.

```text
사용자가 종료를 입력할 때까지 이름과 나이를 입력하세요.
```

이 경우 사용자가 몇 명을 입력할지 모른다.

```text
1명만 입력할 수도 있음
3명 입력할 수도 있음
100명 입력할 수도 있음
바로 종료할 수도 있음
```

그래서 `while(true)`를 사용한다.

정리하면 다음과 같다.

```text
반복 횟수가 정해져 있다
→ for문

반복 횟수는 모르지만 종료 조건이 있다
→ while(true) + break
```

## 5. while(true)는 “다시 물어보기 버튼”이라고 생각하기

`while(true)`를 “무한 반복”이라고만 생각하면 어렵다.

오늘은 이렇게 이해했다.

```text
while(true)
→ 계속 다시 물어보는 구조
```

예를 들어 이름과 나이를 입력받는 코드는 한 바퀴 안에서는 이름과 나이를 한 번씩만 입력받는다.

```text
이름 입력
나이 입력
출력
```

하지만 `while(true)` 안에 이 코드가 들어 있으면,
이 한 세트가 계속 반복된다.

```text
1번째 바퀴:
이름 입력 → 나이 입력 → 출력

2번째 바퀴:
이름 입력 → 나이 입력 → 출력

3번째 바퀴:
이름 입력 → 나이 입력 → 출력

이름에 "종료" 입력:
break → 반복 종료
```

즉, `while(true)`는 이름과 나이를 한 번만 받는 코드를
사용자가 종료할 때까지 계속 반복하게 만든다.

---

## 6. ScannerWhileEx1 코드

오늘 본 코드는 다음과 같다.

```java
package scanner.ex;

import java.util.Scanner;

public class ScannerWhileEx1 {
    public static void main(String[] args) {
        Scanner input = new Scanner(System.in);

        while (true) {
            System.out.print("이름을 입력하세요 (종료를 입력하면 종료): ");
            String name = input.nextLine();

            if (name.equals("종료")) {
                System.out.println("프로그램을 종료합니다.");
                break;
            }

            System.out.print("나이를 입력하세요: ");
            int age = input.nextInt();
            input.nextLine(); // 숫자 입력 후의 줄바꿈 처리

            System.out.println("입력한 이름: " + name + ", 나이: " + age);
        }
    }
}
```

이 코드는 이름과 나이를 계속 입력받다가,
이름에 `"종료"`를 입력하면 프로그램을 끝내는 코드이다.

---

## 7. ScannerWhileEx1 전체 목적

이 코드의 목적은 다음과 같다.

```text
사용자로부터 이름과 나이를 계속 입력받는다.
사용자가 이름에 "종료"라고 입력하면 반복을 끝낸다.
종료가 아니라면 나이를 입력받고 이름과 나이를 출력한다.
```

즉, 이 프로그램은 한 명의 이름과 나이만 입력받고 끝나는 코드가 아니다.

한 바퀴에서는 한 명의 이름과 나이만 입력받지만,
`while(true)` 때문에 그 과정을 계속 반복한다.

---

## 8. ScannerWhileEx1 코드 흐름

코드 흐름은 다음과 같다.

```text
1. Scanner를 만든다.
2. while(true)로 반복을 시작한다.
3. 이름을 입력받는다.
4. 입력한 이름이 "종료"인지 확인한다.
5. "종료"이면 종료 문장을 출력하고 break로 반복을 끝낸다.
6. "종료"가 아니면 나이를 입력받는다.
7. nextLine()으로 남은 줄바꿈을 처리한다.
8. 입력한 이름과 나이를 출력한다.
9. 다시 while문의 처음으로 돌아간다.
```

중요한 부분은 이름을 입력받은 직후에 종료 검사를 한다는 점이다.

```java
if (name.equals("종료")) {
    System.out.println("프로그램을 종료합니다.");
    break;
}
```

사용자가 `"종료"`를 입력했다면,
그건 실제 이름이 아니라 프로그램을 끝내겠다는 신호이다.

그래서 나이를 물어보지 않고 바로 반복을 끝낸다.

## 10. 왜 이름과 나이를 한 번씩 입력받는데 while(true)를 쓰는가?

처음에는 이 부분이 헷갈렸다.

```text
이름과 나이를 한 번씩 입력받는데 왜 무한 반복을 쓰지?
```

정확한 이해는 다음과 같다.

```text
한 바퀴 안에서는 이름과 나이를 한 번씩만 입력받는다.
하지만 그 한 바퀴를 사용자가 "종료"라고 입력할 때까지 계속 반복한다.
```

즉, 한 바퀴의 구조는 다음이다.

```text
이름 입력 → 종료 검사 → 나이 입력 → 출력
```

그리고 `while(true)`는 이 한 바퀴를 계속 반복하게 만든다.

정리하면 다음과 같다.

```text
while(true)는 이름과 나이 입력 세트를 계속 반복하기 위해 사용한다.
사용자가 이름에 "종료"를 입력하면 break로 반복을 끝낸다.
```

## 11. nextInt() 뒤에 nextLine()을 쓰는 이유

다음 코드를 보자.

```java
int age = input.nextInt();
input.nextLine(); // 숫자 입력 후의 줄바꿈 처리
```

`nextInt()`는 숫자만 읽는다.

사용자가 나이를 입력할 때 실제로는 다음처럼 입력한다.

```text
22 + Enter
```

`nextInt()`는 여기서 숫자 `22`만 가져간다.

하지만 Enter로 들어간 줄바꿈은 입력 버퍼에 남아 있을 수 있다.

그 상태에서 다음 반복으로 돌아가서:

```java
String name = input.nextLine();
```

을 실행하면,
남아 있던 Enter를 이름 입력으로 읽어버릴 수 있다.

그래서 `nextInt()` 뒤에 다음 코드를 넣는다.

```java
input.nextLine();
```

이 코드는 남아 있는 줄바꿈을 처리하기 위한 코드이다.

정리하면 다음과 같다.

```text
nextInt()
→ 숫자만 읽는다.

nextLine()
→ 한 줄 전체를 읽는다.

nextInt() 뒤에 남은 Enter를 처리하지 않으면
다음 nextLine()이 이상하게 넘어갈 수 있다.

그래서 nextInt() 뒤에 input.nextLine()을 한 번 더 써서
남은 줄바꿈을 처리한다.
```

## 12. ScannerWhileEx3 코드

두 번째로 공부한 코드는 숫자를 계속 입력받다가,
`-1`이 입력되면 입력을 멈추고 합계와 평균을 출력하는 코드이다.

```java
package scanner.ex;

import java.util.Scanner;

public class ScannerWhileEx3 {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        int sum = 0;
        int count = 0;
        int input = 0;

        System.out.println("숫자를 입력하세요. 입력을 중단하려면 -1을 입력하세요:");

        while (true) {
            input = scanner.nextInt();

            if (input == -1) {
                break;
            }

            sum += input;
            count++;
        }

        double average = (double) sum / count;

        System.out.println("입력한 숫자들의 합계: " + sum);
        System.out.println("입력한 숫자들의 평균: " + average);
    }
}
```

이 코드는 입력한 숫자들의 합계와 평균을 구한다.

단, `-1`은 계산할 숫자가 아니라 입력을 종료하라는 신호이다.

## 13. ScannerWhileEx3 전체 목적

이 코드의 목적은 다음과 같다.

```text
숫자를 계속 입력받는다.
-1을 입력하면 입력을 멈춘다.
-1 이전까지 입력한 숫자들의 합계와 평균을 출력한다.
```

예를 들어 사용자가 다음처럼 입력했다고 하자.

```text
10
20
30
-1
```

그러면 계산 대상은 다음이다.

```text
10, 20, 30
```

`-1`은 계산 대상이 아니다.

`-1`은 종료 신호이다.

그래서 결과는 다음과 같다.

```text
합계 = 10 + 20 + 30 = 60
개수 = 3
평균 = 60 / 3 = 20.0
```

## 14. 변수 역할 정리

```java
int sum = 0;
int count = 0;
int input = 0;
```

각 변수의 역할은 다음과 같다.

```text
sum
→ 입력한 숫자들을 계속 더해두는 변수
→ 합계를 담는 바구니
→ 처음에는 아무것도 더하지 않았기 때문에 0

count
→ 입력한 숫자가 몇 개인지 세는 변수
→ 평균을 구할 때 필요함
→ 처음에는 아무 숫자도 입력하지 않았기 때문에 0

input
→ 사용자가 방금 입력한 숫자를 저장하는 변수
→ 반복문이 돌 때마다 새 입력값으로 바뀜
```

예를 들어 사용자가 `10`을 입력하면:

```text
input = 10
sum = sum + 10
count = count + 1
```

이렇게 된다.

```text

또 이렇게 입력할 수도 있다.

50
-1
```

이 경우 숫자는 5개 입력한 것이다.

즉, 반복 횟수가 정해져 있지 않다.

하지만 끝나는 조건은 알고 있다.

```text
-1이 입력되면 끝난다.
```

그래서 `while(true)`를 사용한다.

```text
while(true)
→ 일단 계속 숫자를 입력받는다.

if (input == -1) break;
→ 그런데 입력값이 -1이면 반복을 끝낸다.
```

## 16. 반복문 내부 흐름

반복문 코드는 다음과 같다.

```java
while (true) {
    input = scanner.nextInt();

    if (input == -1) {
        break;
    }

    sum += input;
    count++;
}
```

이 코드는 다음 순서로 실행된다.

```text
1. 숫자를 하나 입력받는다.
2. 입력한 숫자가 -1인지 확인한다.
3. -1이면 break로 반복문을 끝낸다.
4. -1이 아니면 sum에 더한다.
5. count를 1 증가시킨다.
6. 다시 while문의 처음으로 돌아간다.
```

## 17. 왜 if (input == -1)이 먼저 나오는가?

오늘 가장 중요한 부분이다.

```java
if (input == -1) {
    break;
}

sum += input;
count++;
```

`if (input == -1)`이 `sum += input`보다 먼저 나오는 이유는,
`-1`이 계산할 숫자가 아니기 때문이다.

`-1`은 종료 신호이다.

예를 들어 사용자가 다음처럼 입력했다고 하자.

```text
10
20
-1
```

정상적인 합계는 다음과 같아야 한다.

```text
10 + 20 = 30
```

그런데 만약 `-1`을 먼저 더해버리면 다음처럼 된다.

```text
10 + 20 + (-1) = 29
```

결과가 틀어진다.

그래서 입력을 받자마자 먼저 검사해야 한다.

```text
방금 입력한 값이 -1인가?
→ 맞으면 계산하지 말고 종료한다.

방금 입력한 값이 -1이 아닌가?
→ 그때만 sum에 더하고 count를 증가시킨다.
```

정리하면 다음과 같다.

```text
-1은 숫자가 아니라 종료 신호이다.
그래서 합계에 더하기 전에 먼저 검사한다.
```

## 18. sum += input의 의미

```java
sum += input;
```

이 코드는 다음과 같다.

```java
sum = sum + input;
```

즉, 기존 합계에 방금 입력한 숫자를 더해서 다시 `sum`에 저장한다.

예를 들어 처음 상태가 다음과 같다고 하자.

```text
sum = 0
input = 10
```

그러면:

```text
sum = sum + input
sum = 0 + 10
sum = 10
```

다음에 `20`을 입력하면:

```text
sum = sum + input
sum = 10 + 20
sum = 30
```

이렇게 합계가 계속 누적된다.

## 19. count++의 의미

```java
count++;
```

이 코드는 `count`를 1 증가시키는 코드이다.

다음과 같은 뜻이다.

```java
count = count + 1;
```

`count`는 입력한 숫자의 개수를 세는 변수이다.

예를 들어 사용자가 다음처럼 입력했다고 하자.

```text
10
20
30
-1
```

계산에 들어간 숫자는 다음이다.

```text
10, 20, 30
```

총 3개이다.

그래서 최종적으로 `count`는 3이 되어야 한다.

중요한 점은 `-1`은 count에 포함하지 않는다는 것이다.

```text
-1은 종료 신호이지,
평균을 구할 숫자가 아니다.
```

그래서 `count++`도 `if (input == -1)` 검사 후에 실행된다.

## 21. 평균 계산

반복문이 끝난 후 평균을 계산한다.

```java
double average = (double) sum / count;
```

현재 상태가 다음과 같다면:

```text
sum = 60
count = 3
```

계산은 다음과 같다.

```text
average = 60 / 3
average = 20.0
```

`(double) sum`을 사용하는 이유는 평균은 소수점이 나올 수 있기 때문이다.

예를 들어 다음처럼 입력하면:

```text
10
20
25
-1
```

합계와 개수는 다음이다.

```text
sum = 55
count = 3
```

평균은 다음과 같다.

```text
55 / 3 = 18.333...
```

소수점 결과가 나올 수 있으므로 `double`로 계산한다.

```java
double average = (double) sum / count;
```

## 22. 주석 처리된 두 번째 while문

코드에는 다음 부분도 있었다.

```java
/*
while ((input = scanner.nextInt()) != -1) {
    sum += input;
    count++;
}
*/
```

이 코드는 위의 `while(true)` 방식과 같은 일을 더 짧게 쓴 코드이다.

하지만 초보 입장에서는 어렵게 느껴질 수 있다.

핵심은 이 부분이다.

```java
(input = scanner.nextInt()) != -1
```

이 코드는 숫자를 입력받으면서 동시에 `-1`인지 검사한다.

쉽게 풀면 다음 코드와 같다.

```java
input = scanner.nextInt();

if (input == -1) {
    break;
}
```

즉, 첫 번째 방식은 더 길지만 흐름이 명확하다.

오늘 단계에서는 첫 번째 방식이 더 이해하기 좋다.

```java
while (true) {
    input = scanner.nextInt();

    if (input == -1) {
        break;
    }

    sum += input;
    count++;
}
```

## 23. 오늘 공부한 두 코드의 공통점

오늘 공부한 두 코드는 구조가 비슷하다.

### 이름과 나이 입력 코드

```text
이름을 입력받는다.
이름이 "종료"이면 break 한다.
종료가 아니면 나이를 입력받고 출력한다.
```

### 숫자 합계와 평균 코드

```text
숫자를 입력받는다.
숫자가 -1이면 break 한다.
-1이 아니면 sum에 더하고 count를 증가시킨다.
```

공통 구조는 다음이다.

```java
while (true) {
    입력 받기

    if (종료 조건) {
        break;
    }

    처리하기
}
```

즉, 오늘 공부한 핵심 패턴은 다음이다.

```text
입력 → 종료 검사 → 처리
```

종료 검사보다 처리를 먼저 하면 안 되는 경우가 많다.

왜냐하면 `"종료"`나 `-1`은 실제 데이터가 아니라 프로그램을 끝내기 위한 신호이기 때문이다.

## 24. 코드 독해 방법

오늘 오빠가 말한 것처럼,
나는 문법을 아예 모르는 것이 아니라 코드 독해가 부족한 상태에 가깝다.

문법은 단어 뜻이고,
코드 독해는 문단 흐름을 읽는 것이다.

```text
문법
→ while, if, break, Scanner, nextInt 같은 단어의 뜻

코드 독해
→ 이 코드가 왜 이 순서로 있는지 읽는 것
→ 입력을 먼저 받는 이유
→ 종료 검사를 먼저 하는 이유
→ break가 필요한 이유
→ sum과 count가 왜 필요한지 이해하는 것
```

앞으로 코드를 볼 때는 바로 외우려고 하지 말고,
다음 순서로 읽어야겠다.

```text
1. 이 프로그램의 전체 목적은 무엇인가?
2. 입력은 무엇인가?
3. 출력은 무엇인가?
4. 반복이 필요한 이유는 무엇인가?
5. 반복 종료 조건은 무엇인가?
6. 입력받은 값이 실제 데이터인가, 종료 신호인가?
7. 필요한 변수는 무엇인가?
8. 각 변수는 어떤 역할인가?
9. 코드가 어떤 순서로 실행되는가?
```

## 25. 문제를 코드로 바꾸는 틀

문제를 보면 바로 코드부터 쓰지 말고,
먼저 다음 틀로 정리해야겠다.

```text
문제 목적:
입력:
출력:
반복 필요 여부:
반복 횟수:
종료 조건:
필요한 변수:
처리 순서:
주의할 점:
```

예를 들어 ScannerWhileEx3 문제는 이렇게 정리할 수 있다.

```text
문제 목적:
- 여러 숫자를 입력받고 합계와 평균을 구한다.

입력:
- 숫자 여러 개
- -1을 입력하면 종료

출력:
- 합계
- 평균

반복 필요 여부:
- 필요하다.

반복 횟수:
- 정해져 있지 않다.
- 사용자가 -1을 입력할 때까지 반복한다.

종료 조건:
- input == -1

필요한 변수:
- input: 방금 입력한 숫자
- sum: 숫자들의 합계
- count: 입력한 숫자의 개수
- average: 평균

처리 순서:
1. 숫자를 입력받는다.
2. -1인지 검사한다.
3. -1이면 반복을 종료한다.
4. -1이 아니면 sum에 더한다.
5. count를 1 증가시킨다.
6. 반복이 끝난 뒤 평균을 계산한다.
7. 합계와 평균을 출력한다.

주의할 점:
- -1은 종료 신호이므로 sum과 count에 포함하지 않는다.
```

## 26. 내가 이해한 핵심

오늘 이해한 핵심은 다음이다.

```text
while(true)는 반복 횟수를 모를 때 사용한다.

while(true)는 무조건 영원히 돌리려는 것이 아니라,
반복문 안에서 break로 끝낼 조건을 직접 만들기 위해 사용한다.

입력 반복 문제의 기본 구조는:
입력 → 종료 검사 → 처리

"종료"와 -1은 실제 데이터가 아니라 종료 신호이다.

종료 신호는 계산하거나 출력할 데이터가 아니므로,
처리하기 전에 먼저 검사해야 한다.

sum은 합계를 담는 변수이다.

count는 실제로 입력된 숫자의 개수를 세는 변수이다.

input은 사용자가 방금 입력한 값을 저장하는 변수이다.

평균은 합계 / 개수로 구한다.

평균은 소수점이 나올 수 있으므로 double을 사용한다.

nextInt() 뒤에 nextLine()을 쓰는 이유는 남은 줄바꿈을 처리하기 위해서이다.
```

## 27. 내가 만들려는 프로젝트와의 연결

내가 만들고 싶은 프로젝트는 패션 추천 시스템이다.

오늘 배운 `while(true)`와 `Scanner` 구조는 사용자가 옷 정보를 여러 개 입력하는 기능으로 연결할 수 있다.

예를 들어 사용자가 옷을 몇 개 등록할지 모르는 상황이 있을 수 있다.

```text
상의 이름을 입력하세요. 종료하려면 종료를 입력하세요:
```

사용자가 옷을 1개만 입력할 수도 있고,
10개를 입력할 수도 있다.

이럴 때 반복 횟수를 미리 정할 수 없다.

그래서 `while(true)`를 사용할 수 있다.

```java
while (true) {
    System.out.print("옷 이름을 입력하세요 (종료 입력 시 종료): ");
    String item = scanner.nextLine();

    if (item.equals("종료")) {
        break;
    }

    System.out.println("등록한 옷: " + item);
}
```

추천 점수를 여러 개 입력받고 평균을 구하는 구조도 오늘 배운 코드와 비슷하다.

```java
int sum = 0;
int count = 0;

while (true) {
    System.out.print("추천 점수를 입력하세요 (-1 입력 시 종료): ");
    int score = scanner.nextInt();

    if (score == -1) {
        break;
    }

    sum += score;
    count++;
}

double average = (double) sum / count;
```

이 코드는 패션 추천 시스템에서 다음처럼 연결될 수 있다.

```text
색상 점수 입력
핏 점수 입력
상황 적합도 점수 입력
-1 입력 시 점수 입력 종료
평균 추천 점수 계산
```

즉, 오늘 배운 `while(true)`는 사용자가 몇 개의 데이터를 입력할지 모를 때 매우 중요하다.

## 28. 복학 전 공부 흐름에서의 의미

오늘은 문법을 새로 많이 배웠다기보다,
이미 배운 문법이 실제 문제에서 왜 그렇게 쓰이는지 읽는 연습을 했다.

특히 중요한 것은 다음이다.

```text
코드는 문법만 아는 것으로는 부족하다.
문제의 목적을 읽고,
입력과 출력,

```

오늘은 `while(true)`가 단순히 무한 반복이라는 뜻이 아니라,
사용자가 언제 끝낼지 모르는 입력 반복 문제에서 많이 사용된다는 것을 정리했다.

또 `"종료"`나 `-1` 같은 값은 실제 데이터가 아니라 종료 신호이므로,
처리하기 전에 먼저 검사해야 한다는 점을 배웠다.
