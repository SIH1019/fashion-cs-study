# 2026-06-18 Java 배열 사용 정리

## 1. 오늘 공부한 내용

오늘은 Java 배열 단원에서 `배열 사용` 부분을 공부했다.

앞에서는 다음 코드를 통해 배열을 선언하고 생성하는 방법을 배웠다.

```java
int[] students = new int[5];
```

이 코드는 `int` 값을 담을 수 있는 칸 5개짜리 배열 하나를 생성하고, 실제 배열에 접근할 수 있는 참조값을 `students` 변수에 저장한다.

오늘은 생성한 배열의 각 칸에 값을 넣고, 다시 값을 읽어서 사용하는 방법을 공부했다.

공부한 내용은 다음과 같다.

* 배열 인덱스
* 배열의 인덱스가 0부터 시작하는 이유
* 배열에 값 대입하기
* 배열의 값 읽기
* `students`와 `students[0]`의 차이
* 참조값을 통해 실제 배열에 접근하는 과정
* 배열 인덱스 범위
* `ArrayIndexOutOfBoundsException`
* 기본형과 참조형의 차이
* 배열을 사용할 때 헷갈렸던 참조값 정리

---

## 2. 이전에 공부한 배열 구조 복습

배열을 생성하는 코드는 다음과 같다.

```java
int[] students = new int[5];
```

이 코드는 배열 5개를 만드는 것이 아니다.

```text
int 값을 담을 수 있는 칸 5개짜리 배열 1개를 만든다.
```

개념적으로 보면 다음과 같다.

```text
students
└── x001

x001 ──> [0] [1] [2] [3] [4]
          0   0   0   0   0
```

여기서 역할은 다음과 같다.

```text
new int[5]
→ 칸 5개짜리 실제 배열을 생성한다.

x001
→ 실제 배열에 접근할 수 있는 참조값이다.

students
→ 실제 배열의 참조값 x001을 저장하는 배열 변수이다.
```

`x001`은 실제 Java 코드에 작성하는 값이 아니다.

배열의 참조값을 이해하기 쉽게 표현한 예시이다.

---

## 3. students와 실제 배열의 관계

`students` 변수 안에 배열의 값 5개가 직접 들어있는 것은 아니다.

`students`에는 실제 배열을 찾아갈 수 있는 참조값이 들어있다.

```text
students
└── x001
```

실제 배열은 메모리의 다른 공간에 만들어져 있다.

```text
x001 ──> [0] [1] [2] [3] [4]
          0   0   0   0   0
```

따라서 `students`를 사용하면 Java는 다음 순서로 움직인다고 이해할 수 있다.

```text
1. students에 저장된 참조값을 확인한다.
2. 참조값이 가리키는 실제 배열로 이동한다.
3. 인덱스를 사용해서 원하는 배열의 칸에 접근한다.
```

---

## 4. 배열 인덱스란?

배열의 각 칸을 구분하는 위치 번호를 인덱스라고 한다.

배열을 사용할 때는 대괄호 `[]` 안에 인덱스를 작성한다.

```java
students[0]
students[1]
students[2]
```

각 코드는 다음을 의미한다.

```text
students[0]
→ students가 가리키는 실제 배열의 0번 칸

students[1]
→ students가 가리키는 실제 배열의 1번 칸

students[2]
→ students가 가리키는 실제 배열의 2번 칸
```

여기서 중요한 점은 `students[0]`이 참조값의 첫 번째 부분이라는 뜻이 아니라는 것이다.

```text
students[0]
→ 참조값의 0번째가 아니다.
→ students의 참조값을 따라가서 도착한 실제 배열의 0번 칸이다.
```

---

## 5. 배열은 0번부터 시작한다

Java 배열의 인덱스는 1이 아니라 0부터 시작한다.

```java
int[] students = new int[5];
```

이 배열은 칸이 5개이지만, 인덱스는 다음과 같다.

```text
[0] [1] [2] [3] [4]
```

사용할 수 있는 인덱스는 총 5개이다.

```text
0
1
2
3
4
```

마지막 인덱스는 5가 아니라 4이다.

```text
배열 길이: 5
사용 가능한 인덱스: 0부터 4까지
```

일반적으로 배열의 길이가 `n`이라면 인덱스 범위는 다음과 같다.

```text
0부터 n - 1까지
```

---

## 6. 배열에 값 대입하기

배열의 각 칸에 값을 넣으려면 인덱스를 사용한다.

```java
students[0] = 90;
students[1] = 80;
students[2] = 70;
students[3] = 60;
students[4] = 50;
```

배열에 값을 넣는 방법은 일반 변수와 비슷하다.

일반 변수:

```java
int score = 90;
```

배열:

```java
students[0] = 90;
```

차이점은 배열에서는 대괄호 안에 인덱스를 작성해서 어느 칸에 값을 넣을지 지정한다는 것이다.

---

## 7. students[0] = 90 실행 과정

다음 코드를 자세히 분석하면 다음과 같다.

```java
students[0] = 90;
```

실행 순서:

```text
1. students 변수에 저장된 참조값 x001을 확인한다.
2. x001이 가리키는 실제 배열로 이동한다.
3. 실제 배열에서 0번 인덱스를 찾는다.
4. 0번 칸에 90을 저장한다.
```

개념적으로는 다음처럼 이해할 수 있다.

```java
students[0] = 90;
x001[0] = 90;
```

실제 Java 코드에 `x001`을 작성하는 것은 아니지만, 참조값을 통해 실제 배열에 접근하는 과정을 설명한 것이다.

값을 넣기 전:

```text
students
└── x001

x001 ──> [0] [1] [2] [3] [4]
          0   0   0   0   0
```

값을 넣은 후:

```text
students
└── x001

x001 ──> [0] [1] [2] [3] [4]
         90   0   0   0   0
```

---

## 8. students[1] = 80 실행 과정

다음 코드도 같은 방식으로 동작한다.

```java
students[1] = 80;
```

실행 순서:

```text
1. students 안의 참조값을 확인한다.
2. 참조값을 통해 실제 배열로 이동한다.
3. 실제 배열의 1번 칸을 찾는다.
4. 1번 칸에 80을 저장한다.
```

결과는 다음과 같다.

```text
students
└── x001

x001 ──> [0] [1] [2] [3] [4]
         90  80   0   0   0
```

중요한 점은 `students[1]` 안에 별도의 참조값이 있는 것이 아니라는 것이다.

```text
참조값은 students에 하나만 있다.

students[1]은 students의 참조값을 따라가서
실제 배열의 1번 칸에 접근한 것이다.
```

---

## 9. 배열 전체에 값 대입하기

학생 5명의 점수를 배열에 넣으면 다음과 같다.

```java
int[] students = new int[5];

students[0] = 90;
students[1] = 80;
students[2] = 70;
students[3] = 60;
students[4] = 50;
```

최종 배열의 상태는 다음과 같다.

```text
students
└── x001

x001 ──> [0] [1] [2] [3] [4]
         90  80  70  60  50
```

`students`라는 하나의 배열 변수를 통해서 점수 5개를 관리할 수 있다.

## 10. 배열의 값 읽기

배열에 저장한 값을 읽을 때도 인덱스를 사용한다.

```java
System.out.println(students[0]);
System.out.println(students[1]);
```

이 코드의 실행 과정은 다음과 같다.

```text
1. students의 참조값을 확인한다.
2. 참조값을 따라 실제 배열로 이동한다.
3. 지정한 인덱스의 값을 읽는다.
4. 읽은 값을 출력한다.
```

예를 들어 다음 코드가 있다.

```java
System.out.println("학생1 점수: " + students[0]);
```

개념적으로는 다음처럼 처리된다.

```java
System.out.println("학생1 점수: " + students[0]);
System.out.println("학생1 점수: " + x001[0]);
System.out.println("학생1 점수: " + 90);
```

실행 결과:

```text
학생1 점수: 90
```

---

## 11. 배열 값은 읽어도 사라지지 않는다

일반 변수와 마찬가지로 배열의 값을 읽는다고 해서 값이 없어지는 것은 아니다.

```java
System.out.println(students[0]);
System.out.println(students[0]);
```

`students[0]`에 90이 들어 있다면 두 번 모두 90이 출력된다.

```text
90
90
```

배열의 값은 다른 값을 다시 대입하기 전까지 그대로 유지된다.

---

## 12. 배열의 값 변경하기

배열의 각 칸도 일반 변수처럼 값을 변경할 수 있다.

```java
students[0] = 90;
System.out.println(students[0]);
```

출력:

```text
90
```

이후 같은 칸에 새로운 값을 대입하면 기존 값이 변경된다.

```java
students[0] = 100;
System.out.println(students[0]);
```

출력:

```text
100
```

배열 상태:

```text
변경 전:
[90] [80] [70] [60] [50]

변경 후:
[100] [80] [70] [60] [50]
```

`students[0]`의 기존 값 90은 사라지고 새로운 값 100이 저장된다.

## 13. 배열 사용 전체 코드

```java
package array;

public class Array1Ref1 {
    public static void main(String[] args) {
        int[] students;
        students = new int[5];

        students[0] = 90;
        students[1] = 80;
        students[2] = 70;

        System.out.println("학생4 점수: " + students[3]);
        System.out.println("학생5 점수: " + students[4]);
    }
}
```

```text
학생1 점수: 90
학생2 점수: 80
학생3 점수: 70
학생4 점수: 60
학생5 점수: 50
```

## 14. 코드 구조 분석

### 배열 변수 선언

```java
int[] students;
```

`int` 배열의 참조값을 저장할 수 있는 `students` 변수를 선언한다.

아직 실제 배열은 만들어지지 않았다.

---

### 실제 배열 생성

```java
students = new int[5];
```

칸 5개짜리 `int` 배열을 실제로 만든다.

생성된 배열의 참조값을 `students`에 저장한다.

---

### 배열에 값 대입

```java
students[0] = 90;
```

`students`의 참조값을 따라가서 실제 배열의 0번 칸에 90을 저장한다.

---

### 배열의 값 읽기

```java
System.out.println(students[0]);
```

`students`의 참조값을 따라가서 실제 배열의 0번 칸 값을 읽어 출력한다.

## 15. students와 students[0]의 차이

두 표현은 완전히 다른 역할을 한다.

```java
students
```

뜻:

```text
실제 배열로 가는 참조값을 가진 배열 변수
```

반면 다음 코드는:

```java
students[0]
```

뜻:

```text
students가 가리키는 실제 배열의 0번 칸
```

정리하면 다음과 같다.

```text
students
→ 배열의 위치를 알고 있는 참조 변수

students[0]
→ 실제 배열의 첫 번째 칸

students[1]
→ 실제 배열의 두 번째 칸
```

`students[0]`이 첫 번째 칸이고 `students[1]`이 두 번째 칸인 이유는 배열 인덱스가 0부터 시작하기 때문이다.

## 16. students를 그대로 출력하면?

다음 코드를 실행하면 배열 안의 점수들이 한 번에 출력되지 않는다.

```java
System.out.println(students);
```

출력 결과는 환경에 따라 다음과 비슷하게 나타날 수 있다.

```text
[I@4617c264
```

이유는 `students`가 배열의 값들을 직접 저장하는 것이 아니라 실제 배열을 가리키는 참조값을 가지고 있기 때문이다.

배열의 특정 값을 출력하려면 인덱스를 사용해야 한다.

```java
System.out.println(students[0]);
System.out.println(students[1]);
```

배열 전체 내용을 보기 위해서는 이후에 반복문이나 `Arrays.toString()` 같은 기능을 사용할 수 있지만, 현재 단계에서는 인덱스로 각 칸에 접근하는 방법을 먼저 이해하면 된다.

## 17. 인덱스 범위를 넘으면 안 된다

다음 배열은 길이가 5이다.

```java
int[] students = new int[5];
```

사용 가능한 인덱스는 다음과 같다.

```text
0, 1, 2, 3, 4
```

따라서 다음 코드는 정상적으로 실행된다.

```java
students[4] = 50;
```

하지만 다음 코드는 오류가 발생한다.

```java
students[5] = 100;
```

5번 인덱스는 존재하지 않기 때문이다.

발생하는 오류는 다음과 같다.

```text
ArrayIndexOutOfBoundsException
```

오류 메시지는 다음과 비슷하게 나온다.

```text
Index 5 out of bounds for length 5
```

뜻은 다음과 같다.

```text
배열의 길이는 5인데,
존재하지 않는 5번 인덱스에 접근하려고 했다.
```

## 18. 배열 길이와 마지막 인덱스

배열 길이와 마지막 인덱스는 같지 않다.

```java
int[] students = new int[5];
```

이 배열의 길이는 5이다.

```text
배열 길이 = 5
```

하지만 마지막 인덱스는 4이다.

```text
마지막 인덱스 = 4
```

공식처럼 정리하면 다음과 같다.

```text
마지막 인덱스 = 배열 길이 - 1
```

배열 길이가 5라면:

```text
5 - 1 = 4
```

따라서 마지막 칸은 다음과 같다.

```java
students[4]
```

## 19. 기본형과 참조형

Java의 타입은 크게 기본형과 참조형으로 구분할 수 있다.

### 기본형

기본형은 변수 안에 실제 값을 직접 저장한다.

대표적인 기본형:

```java
int
long
double
boolean
char
```

예시:

```java
int score = 90;
```

구조:

```text
score
└── 90
```

`score` 안에 90이 직접 저장된다.

---

### 참조형

참조형은 실제 데이터가 있는 위치를 가리키는 참조값을 저장한다.

배열은 참조형이다.

```java
int[] students = new int[5];
```

구조:

```text
students
└── x001

x001 ──> [0] [1] [2] [3] [4]
```

`students` 안에 배열 전체가 직접 들어있는 것이 아니라, 실제 배열로 가는 참조값이 들어있다.

---

## 20. 기본형과 참조형 비교

```text
기본형
→ 변수 안에 실제 값을 직접 저장한다.

참조형
→ 변수 안에 실제 데이터로 가는 참조값을 저장한다.
```

예시:

```java
int score = 90;
```

```text
score
└── 90
```

배열 예시:

```java
int[] students = new int[5];
```

```text
students
└── x001

x001 ──> [0] [1] [2] [3] [4]
```

지금 단계에서는 다음 정도로 이해하면 충분하다.

```text
int는 값을 직접 저장한다.
int[]는 배열의 참조값을 저장한다.
```

## 21. 내가 헷갈렸던 내용 정리

처음에는 다음 부분이 헷갈렸다.

```text
students에 실제 배열이 들어있는 것인지
참조값이 들어있는 것인지 헷갈렸다.

students[1]에도 참조값이 따로 있는지 궁금했다.
```

정확한 내용은 다음과 같다.

```text
students에는 실제 배열을 찾아갈 수 있는 참조값 하나가 들어있다.

students[0], students[1], students[2]에
각각 별도의 참조값이 있는 것은 아니다.

students의 참조값을 따라가서 실제 배열에 도착한 뒤,
인덱스를 사용해서 원하는 칸을 선택한다.
```

그림으로 보면 다음과 같다.

```text
students
└── x001  ← 참조값은 여기에 하나 있음

x001 ──> [0] [1] [2] [3] [4]
          ↑   ↑
          │   └── students[1]
          └────── students[0]
```

## 22. 내가 이해한 배열 사용 흐름

```java
int[] students = new int[5];
students[0] = 90;
System.out.println(students[0]);
```

이 코드를 한국어로 바꾸면 다음과 같다.

```text
1. int 값을 저장할 수 있는 칸 5개짜리 배열을 만든다.
2. 배열의 참조값을 students에 저장한다.
3. students의 참조값을 따라 실제 배열로 이동한다.
4. 실제 배열의 0번 칸에 90을 저장한다.
5. 다시 students의 참조값을 따라 배열로 이동한다.
6. 실제 배열의 0번 칸에 있는 90을 읽는다.
7. 90을 출력한다.
```

---

## 23. 배열 사용 핵심 정리

```text
배열의 각 위치 번호를 인덱스라고 한다.

배열의 인덱스는 0부터 시작한다.

new int[5]의 인덱스는 0부터 4까지이다.

students에는 배열의 참조값이 저장되어 있다.

students[0]은 참조값의 첫 번째가 아니다.

students의 참조값을 따라가서 도착한
실제 배열의 0번 칸이다.

배열에 값을 넣을 때:
students[0] = 90;

배열의 값을 읽을 때:
System.out.println(students[0]);

존재하지 않는 인덱스에 접근하면:
ArrayIndexOutOfBoundsException 발생
```

## 24. 내가 만들려는 프로젝트와의 연결

내가 만들고 싶은 프로젝트는 색상, 핏, 기장, 상황 등을 분석해서 옷 조합을 추천하고 설명하는 패션 추천 시스템이다.

배열은 여러 개의 옷 정보나 추천 점수를 한꺼번에 관리할 때 사용할 수 있다.

### 옷 이름 저장

```java
String[] clothes = new String[3];

clothes[0] = "네이비 셔츠";
clothes[1] = "블랙 슬랙스";
clothes[2] = "화이트 스니커즈";
```

이 배열의 상태는 다음과 같다.

```text
[0] 네이비 셔츠
[1] 블랙 슬랙스
[2] 화이트 스니커즈
```

각 옷을 사용할 때는 인덱스로 접근한다.

```java
System.out.println(clothes[0]);
System.out.println(clothes[1]);
```

---

### 추천 점수 저장

```java
int[] scores = new int[3];

scores[0] = 85; // 색상 조합 점수
scores[1] = 70; // 핏 점수
scores[2] = 90; // 상황 적합도 점수
```

이렇게 하면 여러 점수를 하나의 배열로 관리할 수 있다.

```text
scores[0]
→ 색상 조합 점수

scores[1]
→ 핏 점수

scores[2]
→ 상황 적합도 점수
```

나중에는 반복문을 사용해서 배열에 있는 점수를 모두 더하거나 평균을 계산할 수 있다.

배열은 패션 추천 시스템에서 다음과 같은 데이터에 사용할 수 있다.

```text
여러 옷 이름
여러 색상
여러 카테고리
여러 코디 후보
여러 추천 점수
여러 상황별 적합도
```

지금은 학생 점수를 배열에 저장하고 있지만, 같은 구조를 옷 데이터와 추천 점수에 적용할 수 있다.

## 25. 복학 전 공부 흐름에서의 의미

오늘은 배열을 선언하고 생성하는 것에서 더 나아가, 실제 배열의 칸에 값을 넣고 다시 읽는 방법을 공부했다.

특히 단순히 다음 문법만 외우는 것이 아니라:

```java
students[0] = 90;
```

이 코드가 내부적으로 어떻게 움직이는지 이해했다.

```text
students의 참조값을 확인한다.
→ 실제 배열로 이동한다.
→ 인덱스 0번 칸을 찾는다.
→ 그 칸에 90을 저장한다.
```

배열은 앞으로 반복문, 메서드, 객체지향, 자료구조를 공부할 때 계속 사용된다.

지금 참조값과 인덱스 구조를 정확하게 이해해두면 다음 배열 리팩토링과 2차원 배열을 공부할 때 도움이 될 것 같다.
