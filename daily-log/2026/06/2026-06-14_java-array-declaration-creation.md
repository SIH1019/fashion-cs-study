## 2. 배열이 필요한 이유

배열은 같은 타입의 값을 여러 개 저장해야 할 때 사용한다.

예를 들어 학생 5명의 점수를 저장하고 출력한다고 해보자.

배열을 사용하지 않으면 다음처럼 변수를 여러 개 만들어야 한다.

```java
package array;

public class Array1 {
    public static void main(String[] args) {
        int student1 = 90;
        int student2 = 80;
        int student3 = 70;
        int student4 = 60;
        int student5 = 50;

        System.out.println("학생1 점수: " + student1);
        System.out.println("학생2 점수: " + student2);
        System.out.println("학생3 점수: " + student3);
        System.out.println("학생4 점수: " + student4);
        System.out.println("학생5 점수: " + student5);
    }
}
```

학생이 5명이면 변수를 5개 만들면 된다.

하지만 학생이 100명이라면 `student1`, `student2`, `student3`처럼 변수를 100개 만들어야 한다.

이렇게 되면 코드가 너무 길어지고 관리하기 어렵다.

또 반복문을 쓰고 싶어도 변수 이름이 전부 다르기 때문에 반복문으로 처리하기 어렵다.

```text
student1
student2
student3
student4
student5
```

이 변수들은 서로 이름이 다르기 때문에 반복문으로 한 번에 다루기 어렵다.

이런 문제를 해결하기 위해 배열을 사용한다.

---

## 3. 배열이란?

배열은 같은 타입의 변수를 하나로 묶어둔 것이다.

예를 들어 `int` 점수 5개를 저장하고 싶다면,
`int` 변수 5개를 따로 만드는 대신 `int` 배열 하나를 만들 수 있다.

```java
int[] students = new int[5];
```

이 코드는 `int` 값을 담을 수 있는 칸 5개짜리 배열을 만든다.

그림으로 보면 다음과 같다.

```text
[0] [1] [2] [3] [4]
 0   0   0   0   0
```

배열은 여러 개의 값을 하나의 이름으로 관리할 수 있게 해준다.

---

## 4. 배열을 사용한 코드

배열을 사용하면 학생 점수 코드를 다음처럼 바꿀 수 있다.

```java
package array;

public class Array1Ref1 {
    public static void main(String[] args) {
        int[] students; // 배열 변수 선언
        students = new int[5]; // 배열 생성

        students[0] = 90;
        students[1] = 80;
        students[2] = 70;
        students[3] = 60;
        students[4] = 50;

        System.out.println("학생1 점수: " + students[0]);
        System.out.println("학생2 점수: " + students[1]);
        System.out.println("학생3 점수: " + students[2]);
        System.out.println("학생4 점수: " + students[3]);
        System.out.println("학생5 점수: " + students[4]);
    }
}
```

여기서 중요한 코드는 두 줄이다.

```java
int[] students;
students = new int[5];
```

첫 번째 줄은 배열 변수를 선언하는 코드이고,
두 번째 줄은 실제 배열을 생성하는 코드이다.

## 5. 배열 변수 선언

```java
int[] students;
```

이 코드는 `students`라는 배열 변수를 선언하는 것이다.

여기서 `int[]`는 `int 배열`이라는 뜻이다.

즉, `students`는 `int` 값을 여러 개 담는 배열을 가리킬 수 있는 변수이다.

하지만 이 줄만 실행하면 아직 실제 배열이 만들어진 것은 아니다.

```java
int[] students;
```

이 코드는 쉽게 말하면 다음과 같다.

```text
students라는 이름의 배열 변수를 준비한다.
이 변수는 int 배열의 참조값을 저장할 수 있다.
```

주의할 점은 이것이다.

```text
int[] students; 를 썼다고 해서
바로 5칸짜리 배열이 생기는 것은 아니다.
```

배열 변수만 만든 상태이다.

실제 배열은 `new int[5]`를 해야 만들어진다.

---

## 6. 배열 생성

```java
students = new int[5];
```

이 코드는 실제 배열을 생성하는 코드이다.

`new int[5]`의 뜻은 다음과 같다.

```text
int 값을 담을 수 있는 칸 5개짜리 배열 1개를 새로 만든다.
```

여기서 중요한 점은 이것이다.

```text
배열 5개를 만드는 것이 아니다.
칸 5개짜리 배열 1개를 만드는 것이다.
```

그림으로 보면 다음과 같다.

```text
new int[5]

실제 배열 1개
[0] [1] [2] [3] [4]
 0   0   0   0   0
```

---

## 7. new int[5]의 의미

```java
new int[5]
```

이 코드는 메모리 어딘가에 `int` 값을 저장할 수 있는 공간 5개를 만든다.

`int` 하나는 4byte이다.

그래서 `int` 5개를 저장하려면 다음 정도의 공간이 필요하다.

```text
4byte * 5 = 20byte
```

즉, `new int[5]`는 메모리에 20byte 정도의 공간을 확보해서
`int` 값 5개를 저장할 수 있는 배열을 만드는 것이다.

하지만 여기서 중요한 것은 메모리 크기를 외우는 것이 아니다.

중요한 것은 이것이다.

```text
new int[5]는 실제 배열을 만든다.
그 배열은 메모리 어딘가에 만들어진다.
그리고 그 배열에 접근하기 위한 참조값을 반환한다.
```

---

## 8. 배열의 초기값

`new int[5]`로 배열을 만들면 처음 값은 자동으로 초기화된다.

`int` 배열은 숫자 배열이므로 각 칸이 0으로 초기화된다.

```java
int[] students = new int[5];
```

이 상태는 다음과 같다.

```text
students
└── x001

x001 ──> [0] [1] [2] [3] [4]
          0   0   0   0   0
```

아직 값을 직접 넣지 않았지만,
`int` 배열이기 때문에 기본값 0이 들어간다.

자료형에 따른 기본값은 다음처럼 이해하면 된다.

```text
int 배열      → 0
boolean 배열  → false
String 배열   → null
```

---

## 9. 참조값이란?

참조값은 실제 배열을 찾아가기 위한 주소 같은 값이다.

배열은 일반 `int` 변수처럼 변수 안에 값이 직접 들어가는 방식이 아니다.

예를 들어 일반 변수는 이렇게 이해할 수 있다.

```java
int a = 10;
```

```text
a
└── 10
```

`a` 안에 10이 직접 들어있다.

하지만 배열은 다르다.

```java
int[] students = new int[5];
```

이 경우 `students` 안에 배열 5칸이 직접 들어있는 것이 아니다.

`students` 안에는 실제 배열을 찾아가기 위한 참조값이 들어있다.

```text
students
└── x001

x001 ──> 실제 배열
         [0] [1] [2] [3] [4]
          0   0   0   0   0
```

여기서 `x001`은 이해를 돕기 위한 예시이다.

실제로 Java에서 참조값이 `x001`처럼 보이는 것은 아니다.

---

## 10. 왜 참조값을 저장해야 할까?

`new int[5]` 자체에는 이름이 없다.

이 코드는 그냥 메모리 어딘가에 칸 5개짜리 배열을 만드는 것이다.

그런데 이름이 없으면 문제가 생긴다.

```text
배열을 만들긴 만들었는데,
나중에 그 배열을 어떻게 다시 찾아갈까?
```

그래서 Java는 배열을 만들고 나면 그 배열을 찾아갈 수 있는 참조값을 반환한다.

그리고 그 참조값을 배열 변수에 저장해둔다.

```java
int[] students = new int[5];
```

이 코드는 다음처럼 이해할 수 있다.

```text
1. new int[5]로 칸 5개짜리 배열 1개를 만든다.
2. Java가 그 배열을 찾아갈 수 있는 참조값을 반환한다.
3. 그 참조값을 students 변수에 저장한다.
4. 앞으로 students를 통해 그 배열에 접근한다.
```

비유하면 다음과 같다.

```text
new int[5]
→ 5칸짜리 사물함을 만든다.

참조값 x001
→ 그 사물함 위치가 적힌 주소표이다.

students
→ 그 주소표를 보관하는 변수이다.
```

## 11. int[] students = new int[5] 풀어서 보기

```java
int[] students = new int[5];
```

이 한 줄은 강의에서 다음처럼 풀어서 설명한다.

```java
int[] students = new int[5]; // 1. 배열 생성
int[] students = x001;       // 2. new int[5]의 결과로 x001 참조값 반환
students = x001;             // 3. 최종 결과
```

이 코드를 실제 Java 코드처럼 그대로 쓰는 것은 아니다.

`x001`은 참조값을 설명하기 위한 예시이다.

개념적으로는 이렇게 이해하면 된다.

```text
new int[5]가 실제 배열을 만든다.
그 결과로 x001 같은 참조값이 나온다.
students에는 그 참조값이 저장된다.
```

최종 그림은 다음과 같다.

```text
students
└── x001

x001 ──> [0] [1] [2] [3] [4]
          0   0   0   0   0
```

## 12. 내가 이해한 문장 정리

내가 이해한 내용은 다음과 같다.

```text
new int[5]는 배열 5개를 만든다는 뜻이 아니다.
int 값을 담을 수 있는 칸 5개짜리 배열 1개를 만든다는 뜻이다.

배열은 사물함처럼 여러 칸이 있는 큰 공간이다.
그래서 변수 안에 통째로 직접 들어가는 것이 아니라,
그 배열을 찾아가기 위한 참조값을 students에 저장한다.

students라는 배열 변수에는 실제 배열이 아니라,
실제 배열로 가는 참조값 x001이 들어있다.
```

이 문장이 이번 내용의 핵심이다.

## 13. students와 students[0]의 차이

여기서 내가 헷갈렸던 부분은 `students[0]`이다.

처음에는 `students[0]`이 참조값의 첫 번째인가 싶었다.

하지만 정확히는 아니다.

```text
students[0]은 참조값의 첫 번째가 아니다.
students가 들고 있는 참조값을 따라가서 도착한 실제 배열의 0번 칸이다.
```

그림으로 보면 다음과 같다.

```text
students
└── x001

x001 ──> 실제 배열
         [0] [1] [2] [3] [4]
          0   0   0   0   0
```

여기서 참조값은 `students`에 하나만 있다.

```text
students
→ x001 참조값을 들고 있음
```

그리고 `students[0]`, `students[1]`은 각각 실제 배열의 칸이다.

```text
students[0]
→ x001을 따라가서 실제 배열의 0번 칸

students[1]
→ x001을 따라가서 실제 배열의 1번 칸

students[2]
→ x001을 따라가서 실제 배열의 2번 칸
```

즉, `students[1]`에 또 다른 참조값이 있는 것이 아니다.

참조값은 `students`에 하나만 있고,
`students[1]`은 그 참조값을 따라가서 도착한 실제 배열의 1번 칸이다.

## 14. 배열 칸에 값 넣기

```java
students[0] = 90;
students[1] = 80;
```

이 코드는 다음처럼 동작한다.

```text
students 안에 있는 참조값 x001을 본다.
x001이 가리키는 실제 배열로 간다.
그 배열의 0번 칸에 90을 넣는다.
그 배열의 1번 칸에 80을 넣는다.
```

그림으로 보면 다음과 같다.

```text
students
└── x001

x001 ──> [0] [1] [2] [3] [4]
         90  80   0   0   0
```

그래서 다음 코드는:

```java
students[0] = 90;
```

이렇게 해석하면 된다.

```text
students의 참조값을 따라가서,
실제 배열의 0번 칸에 90을 저장해라.
```

다음 코드도 마찬가지이다.

```java
students[1] = 80;
```

```text
students의 참조값을 따라가서,
실제 배열의 1번 칸에 80을 저장해라.
```

## 15. 배열 값 사용하기

배열에 저장한 값을 출력할 때는 다음처럼 쓴다.

```java
System.out.println(students[0]);
System.out.println(students[1]);
```

이 코드도 똑같이 참조값을 따라간다.

```text
students 안에 있는 x001을 본다.
x001이 가리키는 실제 배열로 간다.
그 배열의 0번 칸 값을 읽는다.
그 배열의 1번 칸 값을 읽는다.
```

예를 들어 다음 상태라면:

```text
students
└── x001

x001 ──> [0] [1] [2] [3] [4]
         90  80   0   0   0
```

출력 결과는 다음과 같다.

```text
90
80
```

## 16. students를 그냥 출력하면?

다음 코드를 실행하면 배열의 값들이 출력되는 것이 아니다.

```java
System.out.println(students);
```

이 코드는 `students`가 가지고 있는 참조값 정보를 출력한다.

예를 들면 다음처럼 보일 수 있다.

```text
[I@4617c264
```

여기서 중요한 것은 점수들이 출력되지 않는다는 점이다.

왜냐하면 `students` 안에는 점수 5개가 직접 들어있는 것이 아니라,
배열을 찾아가는 참조값이 들어있기 때문이다.

점수를 출력하고 싶다면 배열의 칸을 지정해야 한다.

```java
System.out.println(students[0]);
System.out.println(students[1]);
```

## 17. 인덱스

배열의 위치 번호를 인덱스라고 한다.

배열은 0번부터 시작한다.

```java
int[] students = new int[5];
```

이 배열은 칸이 5개이다.

하지만 번호는 1부터 5가 아니라 0부터 4까지이다.

```text
[0] [1] [2] [3] [4]
```

그래서 사용할 수 있는 인덱스는 다음과 같다.

```text
0, 1, 2, 3, 4
```

마지막 칸은 `students[4]`이다.

주의할 점은 `students[5]`는 사용할 수 없다는 것이다.

칸이 5개라서 5번 인덱스가 있을 것 같지만,
0부터 시작하기 때문에 마지막은 4번이다.

---

## 18. 인덱스 범위를 넘으면 오류가 난다

```java
students[5] = 100;
```

이 코드는 오류가 난다.

이유는 `new int[5]`로 만든 배열의 인덱스 범위가 0부터 4까지이기 때문이다.

5번 칸은 존재하지 않는다.

이런 오류를 배열 인덱스 범위 초과 오류라고 한다.

```text
ArrayIndexOutOfBoundsException
```

정리하면 다음과 같다.

```text
배열 길이가 5이면 인덱스는 0부터 4까지이다.
배열 길이가 n이면 인덱스는 0부터 n-1까지이다.
```

---

## 19. 기본형과 참조형

이번 배열 단원에서 중요한 개념은 기본형과 참조형이다.

기본형은 변수 안에 값을 직접 저장하는 타입이다.

예를 들어 다음 타입들이 기본형이다.

```java
int
long
double
boolean
```

예시:

```java
int a = 10;
```

이 경우 `a` 안에는 10이 직접 들어있다.

```text
a
└── 10
```

반면 배열은 참조형이다.

```java
int[] students = new int[5];
```

이 경우 `students` 안에는 배열 자체가 직접 들어있는 것이 아니라,
배열을 찾아가는 참조값이 들어있다.

```text
students
└── x001

x001 ──> 실제 배열
         [0] [1] [2] [3] [4]
```

정리하면 다음과 같다.

```text
기본형
→ 변수에 값을 직접 저장한다.

참조형
→ 변수에 실제 데이터로 가는 참조값을 저장한다.
```

## 20. 이번에 내가 헷갈렸던 부분 정리

이번에 내가 헷갈렸던 부분은 다음이었다.

```text
1. students에 배열 자체가 들어있는 줄 알았다.
2. students[0]이 참조값의 첫 번째인지 헷갈렸다.
3. new int[5]가 배열 5개를 만든다는 뜻인지 헷갈렸다.
```

정확한 이해는 다음과 같다.

```text
1. students에는 배열 자체가 아니라 참조값이 들어있다.
2. students[0]은 참조값의 첫 번째가 아니라 실제 배열의 0번 칸이다.
3. new int[5]는 배열 5개가 아니라 칸 5개짜리 배열 1개를 만든다.
```

이렇게 이해하면 된다.

```text
students
→ 배열의 주소표

new int[5]
→ 칸 5개짜리 사물함 하나

students[0]
→ 그 사물함의 0번 칸

students[1]
→ 그 사물함의 1번 칸
```

## 21. 배열의 선언과 생성 전체 핵심 정리

배열은 같은 타입의 변수를 여러 개 묶어두는 것이다.

배열을 사용하면 `student1`, `student2`, `student3`처럼 변수를 여러 개 만들지 않고,
하나의 배열 이름으로 여러 값을 관리할 수 있다.

배열 변수 선언은 다음과 같이 한다.

```java
int[] students;
```

이 코드는 `int` 배열의 참조값을 저장할 수 있는 `students` 변수를 만드는 것이다.

실제 배열 생성은 다음과 같이 한다.

```java
students = new int[5];
```

이 코드는 `int` 값을 담을 수 있는 칸 5개짜리 배열 1개를 생성한다.

그리고 그 배열의 참조값을 `students` 변수에 저장한다.

한 줄로 쓰면 다음과 같다.

```java
int[] students = new int[5];
```

이 코드는 다음 순서로 이해할 수 있다.

```text
1. int 값을 담을 수 있는 칸 5개짜리 배열 1개를 만든다.
2. 그 배열은 메모리 어딘가에 만들어진다.
3. Java가 그 배열을 찾을 수 있는 참조값을 반환한다.
4. students 변수에 그 참조값을 저장한다.
5. students[0], students[1]처럼 참조값을 따라가서 실제 배열의 칸을 사용한다.
```

## 22. 내가 만들려는 프로젝트와의 연결

내가 만들고 싶은 프로젝트는 패션 추천 시스템이다.

이 시스템은 사용자가 입력한 옷 정보를 바탕으로 색상, 핏, 기장, 상황, 분위기 등을 분석해서 옷 조합을 추천하는 방향이다.

배열은 이 프로젝트에서 여러 개의 데이터를 다룰 때 필요하다.

예를 들어 옷 이름 여러 개를 저장할 수 있다.

```java
String[] clothes = new String[5];

clothes[0] = "네이비 셔츠";
clothes[1] = "블랙 슬랙스";
clothes[2] = "화이트 스니커즈";
```

또 색상 점수 여러 개를 저장할 수도 있다.

```java
int[] scores = new int[3];

scores[0] = 80; // 색상 조합 점수
scores[1] = 70; // 핏 점수
scores[2] = 90; // 상황 적합도 점수
```

이렇게 배열을 사용하면 여러 개의 옷 정보나 점수를 하나의 이름으로 관리할 수 있다.

나중에 반복문과 함께 사용하면 배열에 들어있는 옷들을 하나씩 검사할 수 있다.

```java
for (int i = 0; i < clothes.length; i++) {
    System.out.println(clothes[i]);
}
```

패션 추천 시스템에서는 다음과 같은 기능에 배열이 연결될 수 있다.

```text
1. 여러 옷 이름 저장
2. 여러 색상 저장
3. 여러 점수 저장
4. 여러 코디 후보 저장
5. 반복문으로 옷 목록 검사
6. 가장 높은 점수의 코디 찾기
7. 추천 결과 목록 관리
```

지금은 단순히 학생 점수를 배열에 저장하는 예제를 공부하고 있지만,
나중에는 이 구조를 옷 정보와 추천 점수에 적용할 수 있을 것 같다.

## 23. 복학 전 공부 흐름에서의 의미

지금은 휴학 중이고 복학 전까지 Java 기본기를 다시 잡는 중이다.

배열은 앞으로 정말 많이 나오는 개념이다.

특히 배열은 반복문, 메서드, 객체지향으로 넘어가기 전에 꼭 이해해야 하는 기초이다.

이번에 중요한 것은 단순히 `int[] students = new int[5];`를 외우는 것이 아니라,
이 코드가 실제로 어떤 구조를 만드는지 이해하는 것이다.

오늘 이해한 핵심은 다음이다.

```text
new int[5]는 칸 5개짜리 배열 1개를 만든다.
students에는 배열 자체가 아니라 참조값이 들어간다.
students[0]은 참조값의 첫 번째가 아니라 실제 배열의 0번 칸이다.
```

이 정도를 잡고 넘어가면 다음에 배열 사용, 배열 반복문, 2차원 배열을 배울 때 훨씬 덜 헷갈릴 것 같다.
