## 2. 메서드가 필요한 이유

메서드는 반복되는 코드를 줄이기 위해 사용한다.

예를 들어 두 숫자를 더하고 출력하는 기능이 있다고 하자.

먼저 `1 + 2`를 계산하고,
그 다음 `10 + 20`을 계산한다고 하면 다음처럼 코드를 작성할 수 있다.

```java
package method;

public class Method1 {
    public static void main(String[] args) {
        // 계산1
        int a = 1;
        int b = 2;
        System.out.println(a + "+" + b + " 연산 수행");
        int sum1 = a + b;
        System.out.println("결과1 출력: " + sum1);

        // 계산2
        int x = 10;
        int y = 20;
        System.out.println(x + "+" + y + " 연산 수행");
        int sum2 = x + y;
        System.out.println("결과2 출력: " + sum2);
    }
}
```

이 코드는 실행은 되지만 문제가 있다.

비슷한 코드가 반복된다.

```java
System.out.println(a + "+" + b + " 연산 수행");
int sum1 = a + b;
System.out.println("결과1 출력: " + sum1);
```

```java
System.out.println(x + "+" + y + " 연산 수행");
int sum2 = x + y;
System.out.println("결과2 출력: " + sum2);
```

숫자만 다르고 구조는 거의 같다.

---

## 3. 반복되는 코드의 문제

반복되는 코드는 처음에는 괜찮아 보여도,
코드가 길어질수록 문제가 생긴다.

예를 들어 두 수를 더하는 코드가 2번이 아니라 100번 반복된다고 생각해보자.

그러면 다음 문제가 생긴다.

```text
1. 코드가 너무 길어진다.
2. 같은 로직을 계속 복사해야 한다.
3. 수정할 일이 생기면 여러 곳을 다 고쳐야 한다.
4. 실수할 가능성이 높아진다.
5. 코드가 무엇을 하는지 한눈에 보기 어렵다.
```

그래서 반복되는 코드는 하나로 묶어두는 것이 좋다.

이때 사용하는 것이 메서드이다.

---

## 4. 메서드란?

메서드는 특정 기능을 수행하는 코드 묶음이다.

쉽게 말하면 다음과 같다.

```text
메서드
→ 자주 쓰는 코드를 따로 묶어둔 것
→ 이름을 붙여두고 필요할 때 호출해서 사용한다
```

예를 들어 두 숫자를 더하는 기능을 `add`라는 이름의 메서드로 만들 수 있다.

```java
public static int add(int a, int b) {
    System.out.println(a + "+" + b + " 연산 수행");
    int sum = a + b;
    return sum;
}
```

이 메서드는 두 숫자를 받아서 더한 다음,
그 결과를 다시 돌려준다.

## 5. 메서드 사용 코드

반복되는 더하기 코드를 메서드로 바꾸면 다음처럼 작성할 수 있다.

```java
package method;

public class Method1Ref {
    public static void main(String[] args) {
        int sum1 = add(1, 2);
        System.out.println("결과1 출력: " + sum1);

        int sum2 = add(10, 20);
        System.out.println("결과2 출력: " + sum2);
    }

    public static int add(int a, int b) {
        System.out.println(a + "+" + b + " 연산 수행");
        int sum = a + b;
        return sum;
    }
}
```

이 코드에서는 더하기 기능을 `add()` 메서드로 따로 분리했다.

그래서 `main()`에서는 더하기가 필요할 때마다 `add()`를 호출하면 된다.

```java
int sum1 = add(1, 2);
int sum2 = add(10, 20);
```

이렇게 하면 같은 계산 코드를 여러 번 복사하지 않아도 된다.

## 6. main 메서드와 add 메서드

위 코드에는 메서드가 2개 있다.

```java
public static void main(String[] args)
```

```java
public static int add(int a, int b)
```

`main()` 메서드는 프로그램이 시작되는 곳이다.

`add()` 메서드는 두 숫자를 더하는 기능을 담당한다.

정리하면 다음과 같다.

```text
main()
→ 프로그램 시작점
→ add() 메서드를 호출해서 사용함

add()
→ 두 숫자를 더하는 기능
→ 계산 결과를 return으로 돌려줌
```

## 7. 메서드 호출

메서드를 사용하려면 메서드 이름을 적고,
필요한 값을 괄호 안에 넣어 호출한다.

```java
add(1, 2)
```

이 코드는 `add` 메서드에게 숫자 1과 2를 전달하면서 실행하라는 뜻이다.

다음 코드도 마찬가지이다.

```java
add(10, 20)
```

이 코드는 `add` 메서드에게 숫자 10과 20을 전달하면서 실행하라는 뜻이다.

메서드를 호출하면,
Java는 해당 메서드로 이동해서 메서드 본문을 실행한다.

## 8. add(1, 2)의 실행 흐름

다음 코드가 있다고 하자.

```java
int sum1 = add(1, 2);
```

이 코드는 바로 `sum1`에 값이 들어가는 것이 아니다.

먼저 `add(1, 2)` 메서드를 호출한다.

실행 흐름은 다음과 같다.

```text
1. main()에서 add(1, 2)를 호출한다.
2. add 메서드로 이동한다.
3. a에는 1이 들어간다.
4. b에는 2가 들어간다.
5. int sum = a + b; 를 실행한다.
6. sum은 3이 된다.
7. return sum; 으로 3을 돌려준다.
8. add(1, 2)는 결과값 3으로 바뀐다.
9. int sum1 = 3; 이 된다.
10. sum1에 3이 저장된다.
```

코드로 보면 다음 흐름이다.

```java
int sum1 = add(1, 2);
```

```java
public static int add(int a, int b) {
    System.out.println(a + "+" + b + " 연산 수행");
    int sum = a + b;
    return sum;
}
```

`add(1, 2)`가 실행되면:

```text
a = 1
b = 2
sum = 1 + 2
sum = 3
return 3
```

결국 다음처럼 이해할 수 있다.

```java
int sum1 = add(1, 2);
int sum1 = 3;
```

즉, 메서드 호출이 끝나면 메서드가 반환한 값으로 바뀐다.

## 9. add(10, 20)의 실행 흐름

다음 코드도 같은 방식이다.

```java
int sum2 = add(10, 20);
```

실행 흐름은 다음과 같다.

```text
1. main()에서 add(10, 20)를 호출한다.
2. add 메서드로 이동한다.
3. a에는 10이 들어간다.
4. b에는 20이 들어간다.
5. int sum = a + b; 를 실행한다.
6. sum은 30이 된다.
7. return sum; 으로 30을 돌려준다.
8. add(10, 20)는 결과값 30으로 바뀐다.
9. int sum2 = 30; 이 된다.
10. sum2에 30이 저장된다.
```

정리하면 다음과 같다.

```java
int sum2 = add(10, 20);
int sum2 = 30;
```

## 10. 메서드 정의

메서드는 다음과 같은 형태로 정의한다.

```java
public static int add(int a, int b) {
    System.out.println(a + "+" + b + " 연산 수행");
    int sum = a + b;
    return sum;
}
```

전체 구조는 다음과 같다.

```java
제어자 반환타입 메서드이름(매개변수 목록) {
    메서드 본문
}
```

`add` 메서드를 구조로 나누면 다음과 같다.

```text
public static
→ 제어자

int
→ 반환 타입

add
→ 메서드 이름

int a, int b
→ 매개변수

중괄호 안 코드
→ 메서드 본문
```

## 11. 제어자 public static

```java
public static
```

이 부분은 제어자이다.

지금 단계에서는 깊게 이해하지 않아도 된다.

현재 강의 범위에서는 메서드를 만들 때 일단 다음처럼 작성한다고 생각하면 된다.

```java
public static
```

나중에 객체지향을 배우면 `public`, `static`의 의미를 더 자세히 배우게 된다.

지금은 다음 정도로 이해한다.

```text
public static
→ 지금은 메서드를 만들 때 붙이는 기본 형식으로 생각한다.
```

## 12. 반환 타입 int

```java
public static int add(int a, int b)
```

여기서 `int`는 반환 타입이다.

반환 타입은 메서드가 실행된 후 돌려주는 값의 타입을 의미한다.

`add` 메서드는 두 숫자를 더한 결과를 돌려준다.

두 숫자가 정수이고 결과도 정수이므로 반환 타입은 `int`이다.

```java
return sum;
```

여기서 `sum`은 `int` 타입이다.

그래서 메서드 앞에도 `int`라고 적는다.

```text
반환 타입 int
→ 이 메서드는 int 값을 돌려준다.
```

## 13. 메서드 이름 add

```java
add
```

`add`는 메서드 이름이다.

메서드 이름은 그 기능을 나타내도록 짓는 것이 좋다.

여기서는 두 숫자를 더하는 기능이므로 `add`라는 이름을 사용했다.

```text
add
→ 더하다
→ 두 숫자를 더하는 메서드
```

메서드를 사용할 때도 이 이름으로 호출한다.

```java
add(1, 2);
add(10, 20);
```

## 14. 매개변수 int a, int b

```java
public static int add(int a, int b)
```

여기서 괄호 안에 있는 `int a, int b`가 매개변수이다.

매개변수는 메서드가 외부에서 전달받은 값을 담는 변수이다.

```text
매개변수
→ 메서드가 입력받은 값을 저장하는 변수
```

예를 들어 다음처럼 호출하면:

```java
add(1, 2)
```

값이 이렇게 들어간다.

```text
a = 1
b = 2
```

다음처럼 호출하면:

```java
add(10, 20)
```

값이 이렇게 들어간다.

```text
a = 10
b = 20
```

즉, `a`와 `b`는 호출할 때 전달한 값을 받아서 메서드 안에서 사용하는 변수이다.

## 15. 인수와 매개변수

이 부분은 헷갈리기 쉽다.

```java
int sum1 = add(1, 2);
```

여기서 `1`, `2`는 메서드에 전달하는 값이다.

이런 값을 인수 또는 인자라고 한다.

```text
인수 또는 인자
→ 메서드를 호출할 때 실제로 넣는 값
→ add(1, 2)의 1과 2
```

반면 메서드 정의 부분의 `a`, `b`는 매개변수이다.

```java
public static int add(int a, int b)
```

```text
매개변수
→ 메서드가 전달받은 값을 저장하는 변수
→ int a, int b
```

정리하면 다음과 같다.

```text
add(1, 2)
→ 1, 2는 인수

public static int add(int a, int b)
→ a, b는 매개변수
```

쉽게 말하면:

```text
인수
→ 보내는 값

매개변수
→ 받는 변수
```

## 16. 메서드 본문

메서드 본문은 중괄호 `{}` 안에 있는 코드이다.

```java
{
    System.out.println(a + "+" + b + " 연산 수행");
    int sum = a + b;
    return sum;
}
```

이 부분이 실제로 메서드가 수행하는 작업이다.

`add` 메서드의 본문은 다음 일을 한다.

```text
1. 어떤 연산을 수행하는지 출력한다.
2. a와 b를 더해서 sum에 저장한다.
3. sum을 return으로 돌려준다.
```

즉, 메서드를 호출하면 이 본문이 위에서 아래로 순서대로 실행된다.

## 17. return의 의미

```java
return sum;
```

`return`은 메서드의 실행 결과를 호출한 곳으로 돌려주는 역할을 한다.

`add` 메서드에서는 `sum` 값을 돌려준다.

```java
int sum = a + b;
return sum;
```

예를 들어 `a = 1`, `b = 2`라면:

```text
sum = 3
return sum
return 3
```

그래서 `add(1, 2)`의 결과는 3이 된다.

```java
int sum1 = add(1, 2);
int sum1 = 3;
```

정리하면 다음과 같다.

```text
return
→ 메서드의 결과를 호출한 곳으로 돌려준다.
```

## 18. 메서드 호출은 결과값으로 바뀐다

메서드 호출이 끝나면,
그 자리는 메서드가 반환한 값으로 바뀐다고 생각하면 된다.

```java
int sum1 = add(5, 10);
```

`add(5, 10)`이 실행되면 결과는 15이다.

그래서 개념적으로는 다음과 같다.

```java
int sum1 = add(5, 10);
int sum1 = 15;
```

이렇게 메서드가 반환한 결과값이 변수에 저장된다.

이 부분이 메서드 사용에서 가장 중요하다.

```text
메서드 호출
→ 메서드 본문 실행
→ return으로 결과 반환
→ 호출한 자리가 반환값으로 바뀜
```

## 19. 메서드를 사용하면 좋은 점

메서드를 사용하면 반복되는 코드를 줄일 수 있다.

메서드 사용 전에는 비슷한 계산 코드를 계속 작성해야 했다.

```java
int a = 1;
int b = 2;
int sum1 = a + b;

int x = 10;
int y = 20;
int sum2 = x + y;
```

하지만 메서드를 사용하면 더하기 기능을 한 번만 만들어두고,
필요할 때 호출해서 사용할 수 있다.

```java
int sum1 = add(1, 2);
int sum2 = add(10, 20);
```

장점은 다음과 같다.

```text
1. 중복 코드가 줄어든다.
2. 코드가 짧아진다.
3. 코드의 의미가 더 잘 보인다.
4. 수정할 때 한 곳만 고치면 된다.
5. 기능별로 코드를 나눌 수 있다.
```

## 20. 메서드 사용 전과 사용 후 비교

### 메서드 사용 전

```java
int a = 1;
int b = 2;
System.out.println(a + "+" + b + " 연산 수행");
int sum1 = a + b;
System.out.println("결과1 출력: " + sum1);

int x = 10;
int y = 20;
System.out.println(x + "+" + y + " 연산 수행");
int sum2 = x + y;
System.out.println("결과2 출력: " + sum2);
```

비슷한 코드가 반복된다.

---

### 메서드 사용 후

```java
int sum1 = add(1, 2);
System.out.println("결과1 출력: " + sum1);

int sum2 = add(10, 20);
System.out.println("결과2 출력: " + sum2);
```

더하기 기능은 아래 메서드에 따로 모아둔다.

```java
public static int add(int a, int b) {
    System.out.println(a + "+" + b + " 연산 수행");
    int sum = a + b;
    return sum;
}
```

이렇게 하면 `main()`에서는 흐름만 깔끔하게 볼 수 있다.

```text
1. add(1, 2)를 호출한다.
2. 결과를 sum1에 저장한다.
3. 출력한다.
4. add(10, 20)를 호출한다.
5. 결과를 sum2에 저장한다.
6. 출력한다.
```

## 21. 내가 헷갈릴 수 있는 부분 정리

이번 범위에서 헷갈릴 수 있는 부분은 다음이다.

```text
1. 메서드는 왜 필요한가?
2. add(1, 2)는 무슨 뜻인가?
3. int a, int b는 어디서 오는가?
4. return sum은 왜 필요한가?
5. add(1, 2)가 어떻게 sum1에 저장되는가?
```

정확한 이해는 다음과 같다.

```text
메서드는 반복되는 코드를 줄이기 위해 사용한다.
```

## 22. 한 줄씩 읽는 연습

다음 코드를 한 줄씩 읽어보자.

```java
int sum1 = add(1, 2);
```

읽는 방법:

```text
add 메서드에 1과 2를 전달해서 호출한다.
add 메서드가 두 수를 더한다.
결과 3을 반환한다.
반환된 3을 sum1에 저장한다.
```

다음 코드도 읽어보자.

```java
public static int add(int a, int b) {
    System.out.println(a + "+" + b + " 연산 수행");
    int sum = a + b;
    return sum;
}
```

읽는 방법:

```text
add라는 메서드를 만든다.
이 메서드는 int 값 2개를 입력받는다.
입력받은 값은 a와 b에 저장된다.
a와 b를 더해서 sum에 저장한다.
sum을 호출한 곳으로 돌려준다.
```

## 24. 내가 만들려는 프로젝트와의 연결

내가 만들고 싶은 프로젝트는 패션 추천 시스템이다.

이 시스템은 사용자가 입력한 옷 정보를 바탕으로 색상, 핏, 기장, 상황, 분위기 등을 분석해서 옷 조합을 추천하는 방향이다.

메서드는 이 프로젝트에서 기능을 나누는 데 중요하다.

예를 들어 패션 추천 시스템에서는 다음 기능들이 필요할 수 있다.

```text
1. 색상 점수 계산
2. 핏 점수 계산
3. 상황 적합도 점수 계산
4. 전체 평균 점수 계산
5. 추천 결과 출력
```

이 기능들을 전부 `main()` 안에 작성하면 코드가 너무 길어진다.

그래서 메서드로 나눌 수 있다.

예를 들어 색상 점수를 계산하는 메서드는 다음처럼 만들 수 있다.

```java
public static int getColorScore(String color) {
    if (color.equals("black")) {
        return 90;
    } else if (color.equals("navy")) {
        return 80;
    } else {
        return 60;
    }
}
```

평균 점수를 계산하는 메서드는 다음처럼 만들 수 있다.

```java
public static double getAverageScore(int colorScore, int fitScore, int moodScore) {
    int total = colorScore + fitScore + moodScore;
    return (double) total / 3;
}
```

나중에 패션 추천 시스템을 만들 때는 `main()`에 모든 코드를 다 넣는 것이 아니라,
기능별로 메서드를 나누는 연습이 필요하다.

```text
색상 판단 메서드
핏 판단 메서드
상황 판단 메서드
평균 계산 메서드
추천 출력 메서드
```

지금 배우는 `add()` 메서드는 단순히 숫자를 더하는 예제이지만,
나중에는 추천 로직을 기능별로 나누는 기본기가 된다.

## 25. 복학 전 공부 흐름에서의 의미

지금은 Java 기본기를 다시 잡는 과정이다.

배열까지는 여러 값을 저장하고 반복문으로 처리하는 방법을 배웠다.

메서드부터는 코드를 기능 단위로 나누는 연습이 시작된다.

이번에 중요한 것은 다음이다.

```text
반복되는 코드는 메서드로 뺄 수 있다.

메서드 호출 결과는 반환값으로 바뀐다.
```

이 내용을 이해하면 앞으로 문제를 풀 때도 코드를 더 깔끔하게 나눌 수 있을 것 같다.
